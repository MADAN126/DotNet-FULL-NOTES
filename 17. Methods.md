# Why Do Methods Exist?

Before learning syntax like:

```csharp
static void Greet()
{
    Console.WriteLine("Hello");
}
```

ask a deeper question:

> Why do programming languages need methods?

Imagine building an application without methods.

Suppose you need to validate an email address in 15 different places.

Without methods:

```csharp
// Validation Logic
...

// Same Validation Logic Again
...

// Same Validation Logic Again
...
```

Problems:

```text
Massive duplication
Hard to maintain
Easy to introduce bugs
Changes must be made everywhere
Code becomes unreadable
```

Now imagine:

```text
Calculate taxes
Authenticate users
Send emails
Save orders
Generate reports
Validate forms
Process payments
```

Without methods:

> Every program would become one gigantic block of code.

Methods exist because:

> Programs need a way to group related instructions into reusable, meaningful units.

# What Is a Method?

A method is:

> A named block of code that performs a specific task.

Example:

```csharp
static void SayHello()
{
    Console.WriteLine("Hello");
}
```

---

# Why Do Methods Exist?

Methods solve several problems.

---

## Problem 1: Duplication

Without methods:

```csharp
Console.WriteLine("Welcome");
Console.WriteLine("Welcome");
Console.WriteLine("Welcome");
```

With methods:

```csharp
DisplayWelcome();
DisplayWelcome();
DisplayWelcome();
```

---

## Problem 2: Readability

Bad:

```csharp
// 500 lines of code
```

Good:

```csharp
ValidateInput();
CalculateTotal();
SaveOrder();
SendConfirmation();
```

---

## Problem 3: Maintainability

Change logic once:

```text
Every caller benefits.
```

---

## Problem 4: Reusability

Write once.

Use everywhere.

# Method Overloading

One of C#'s most powerful features.

---

# What Is Method Overloading?

Multiple methods with:

```text
Same name
Different parameters
```

---

# Why Does It Exist?

Because similar operations often work with different inputs.

---

# Example

```csharp

{
    return a + b;
}

{
    return a + b;
}
```

---

# Mental Model

```text

Different Versions
```

# How Does C# Decide?


# Pass By Value

Default behavior.

Example:

```csharp

{
    x = 50;
}

```

Output:

```text

```

---

# Why?

Execution:

```text
Copy value
↓
Modify copy
↓
Original unchanged
```

---

# Pass By Reference (ref)

Example:

```csharp

{
    x = 50;
}

Change(ref number);
```

Output:

```text
50
```

# Recursion

---

# What Is Recursion?

A method calling itself.

---

# Example

```csharp
static void Countdown(int n)
{
    if(n == 0)
        return;

    Console.WriteLine(n);

    Countdown(n - 1);
}
```

Output:

```text
3
2
1
```

# Interview Perspective

## What is a method?

Answer:

A named block of reusable code that performs a specific task.

---

## Difference between parameters and arguments?

Answer:

Parameters are variables defined by the method. Arguments are actual values passed during method calls.

---

## What is method overloading?

Answer:

Defining multiple methods with the same name but different parameter lists.

---

## What is recursion?

Answer:

A method calling itself until a base condition stops further calls.

---

## Difference between pass by value and ref?

Answer:

Pass by value copies data. `ref` allows modification of the original variable.

---

## What does return do?

Answer:

It sends a value back and immediately exits the method.

---

# Best Practices Followed by Experienced Developers

- Give methods meaningful names.
- Keep methods small and focused.
- Follow the Single Responsibility Principle.
- Prefer returning values over modifying state unnecessarily.
- Avoid long parameter lists.
- Use overloading thoughtfully.
- Write methods that express business intent clearly.
- Avoid deep recursion unless necessary.
- Make methods easy to test.

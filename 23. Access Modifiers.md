# C# ACCESS MODIFIERS

---

# Why Do Access Modifiers Exist?

Before learning keywords like:

```csharp
public
private
protected
internal
protected internal
private protected
```

ask a deeper question:

> Why should some parts of a class be hidden while others are exposed?

Imagine building a banking application.

```csharp
class BankAccount
{
    public decimal Balance;
}
```

Anyone can do:

```csharp
account.Balance = -1000000;
```

Problems:

```text
Invalid state
Broken business rules
Security issues
Difficult debugging
Unpredictable behavior
```

Now imagine a car.

Should users directly manipulate:

```text
Fuel injection timing?
Engine combustion sequence?
Brake pressure calculations?
```

No.

Users should interact through:

```text
Start()
Accelerate()
Brake()
```

not internal machinery.

Access modifiers exist because:

> Objects should expose only what others need while protecting their internal implementation.

---

# Mental Model

Think of a smartphone.

Visible to users:

```text
Camera App
Messages
Phone App
Settings
```

Hidden:

```text
Operating system internals
Security keys
Kernel processes
Encryption mechanisms
```

You interact with:

```text
Public features.
```

The phone protects:

```text
Private implementation.
```

Classes work the same way.

---

# What Are Access Modifiers?

Access modifiers are:

> Keywords that determine where class members can be accessed.

They control:

```text
Visibility
Accessibility
Encapsulation
Protection
API boundaries
```

---

# Why Are They Important?

Without access modifiers:

```text
Everything is accessible.
Everything can be modified.
Everything becomes fragile.
```

Access modifiers help achieve:

```text
Encapsulation
Security
Maintainability
Reduced coupling
Cleaner APIs
```

---

# The Main Access Modifiers

C# provides six major access modifiers:

```text
public
private
protected
internal

```

# 1. public

---

# What Is It?

Accessible from anywhere.

---

# Mental Model

Think:

```text
Public Park
```

Anyone can enter.

---

# Example

```csharp
class Car
{
    public string Brand;
}
```

Usage:

```csharp
Car car = new Car();

car.Brand = "Tesla";

Console.WriteLine(car.Brand);
```

---

# Where Can It Be Accessed?

```text
Same class
Same project
Derived classes
Other projects
Everywhere
```

---

# Why Use public?

When something is intended to be part of the class's public API.

---

# Real-World Examples

```csharp
Console.WriteLine()

Math.Sqrt()

DateTime.Now
```

---

# Advantages

```text
Easy accessibility
Simple API exposure
Supports collaboration
```

---

# Disadvantages

```text
Exposes implementation
Harder to change later
Can allow misuse
```

---

# When To Use

```text
Methods intended for consumers.
Properties meant to be used externally.
Public APIs.
```

---

# When To Avoid

```text
Internal implementation details.
Sensitive data.
```

---

# 2. private

---

# What Is It?

Accessible only inside the same class.

---

# Mental Model

Think:

```text
Personal Diary
```

Only the owner can access it.

---

# Example

```csharp
class BankAccount
{
    private decimal balance;
}
```

---

# Usage

Inside class:

```csharp
balance = 1000;
```

Outside class:

```csharp
account.balance = 1000;
```

Error.

---

# Why Does private Exist?

To protect internal state.

---

# Example

Bad:

```csharp
public decimal Balance;
```

Good:

```csharp
private decimal balance;
```

---

# Advantages

```text
Strong encapsulation
Prevents misuse
Easier maintenance
Safer design
```

---

# Disadvantages

```text
Requires controlled access mechanisms.
```

Usually solved using properties.

---

# Best Practice

> Default to private unless something truly needs broader access.

---

# 3. protected

---

# What Is It?

Accessible:

```text
Inside the same class
+
Derived classes
```

---

# Mental Model

Think:

```text
Family Inheritance
```

Parents and children can access it.

Strangers cannot.

---

# Example

```csharp
class Animal
{
    protected string Name;
}

class Dog : Animal
{
    public void SetName()
    {
        Name = "Rocky";
    }
}
```

---

# Outside Access

```csharp
Dog dog = new Dog();

dog.Name = "Rocky";
```

Error.

---

# Why Use protected?

When derived classes need access.

---

# Real-World Usage

Framework design.

Base classes exposing extension points.

---

# Advantages

```text
Supports inheritance
Encourages extensibility
Protects external access
```

---

# Disadvantages

```text
Can tightly couple inheritance hierarchies.
```

---

# 4. internal

---

# What Is It?

Accessible only within the same assembly (project).

---

# Mental Model

Think:

```text
Office Building
```

Employees can enter.

People from outside companies cannot.

---

# Example

```csharp
internal class Helper
{
}
```

---

# Accessible From

```text
Same project → Yes
Different project → No
```

---

# Why Use internal?

To hide implementation details from other assemblies.

---

# Real-World Usage

Libraries.

Utility classes.

Infrastructure code.

---

# Advantages

```text
Cleaner public APIs
Assembly-level protection
Reduces external dependencies
```

---

# Disadvantages

```text
Less reusable outside assembly.
```

---

# 5. protected internal

---

# What Is It?

Accessible from:

```text
Same assembly
OR
Derived classes in other assemblies
```

---

# Mental Model

Think:

```text
Employees
OR
Family Members
```

Either group can enter.

---

# Example

```csharp
protected internal void Process()
{
}
```

---

# Accessibility

```text
Same project → Yes

Derived classes outside project → Yes

Unrelated external classes → No
```

---

# Why Use It?

Framework extensibility.

Library design.

---

# 6. private protected

---

# What Is It?

Accessible from:

```text
Same assembly
AND
Derived classes
```

---

# Mental Model

Think:

```text
Children living inside the family house.
```

Outside relatives cannot access.

---

# Accessibility

```text
Same class → Yes

Derived classes in same assembly → Yes

Derived classes outside assembly → No

Other classes → No
```

---

# Why Use It?

Very controlled inheritance scenarios.

Advanced framework design.

---

# Accessibility Comparison Table

| Modifier | Same Class | Same Project | Derived Class | Other Projects |
|-----------|------------|--------------|---------------|----------------|
| private | Yes | No | No | No |
| private protected | Yes | Yes* | Yes* | No |
| protected | Yes | No | Yes | Yes |
| internal | Yes | Yes | Yes** | No |
| protected internal | Yes | Yes | Yes | Yes* |
| public | Yes | Yes | Yes | Yes |

Notes:

```text
* Under specific conditions.
** Only if inside same assembly.
```

# Access Modifiers and Encapsulation

One of OOP's core principles.

---

# Encapsulation Means

> Hide internal implementation and expose only what is necessary.

---

# Example

Bad:

```csharp
class Account
{
    public decimal Balance;
}
```

---

Good:

```csharp
class Account
{
    private decimal balance;

    public decimal Balance
    {
        get
        {
            return balance;
        }
    }

    public void Deposit(decimal amount)
    {
        if (amount > 0)
        {
            balance += amount;
        }
    }
}
```

---

# Why Better?

Users can:

```text
Read balance.
Deposit money.
```

Users cannot:

```text
Corrupt balance.
```

---



# Top-Level Classes Default To

```text

```

Example:

```csharp

{
}
```

Equivalent to:

```csharp

{
}
```

# Hidden Behaviors and Edge Cases

---

# private Applies Per Class

Objects of the same class can access each other's private members.

Example:

```csharp
class Person
{
    private int age;

    public void CopyAge(Person other)
    {
        age = other.age;
    }
}
```

Valid.

---

# protected Does NOT Mean Same Project

Only inheritance matters.

---

# internal Is Assembly-Based

Not namespace-based.

---

# Public APIs Are Hard To Change

Once exposed:

```text
Consumers depend on them.
```

# Interview Perspective

## What are access modifiers?

Answer:

Keywords that control where classes and members can be accessed.

---

## What is the difference between public and private?

Answer:

Public members are accessible from anywhere, while private members are accessible only within the same class.

---

## What is protected?

Answer:

Accessible within the same class and derived classes.

---

## What is internal?

Answer:

Accessible only within the same assembly.

---

## What is encapsulation?

Answer:

The practice of hiding internal details and exposing only necessary functionality.

---

## What is the default access modifier for class members?

Answer:

Private.

---

## What is the default access modifier for top-level classes?

Answer:

Internal.

---

# Best Practices Followed by Experienced Developers

- Default to `private`.
- Expose the smallest possible surface area.
- Use properties instead of public fields.
- Make APIs intentionally public.
- Use `internal` for implementation details.
- Use `protected` only when inheritance genuinely requires it.
- Avoid exposing sensitive information.
- Think carefully before making something public because changing public APIs later is difficult.

# Final Mental Model

Think of access modifiers as security levels.

```text
Class Members
↓
Need Protection?
↓
Choose Appropriate Visibility

private
↓
Only Me

protected
↓
Me + My Children

internal
↓
My Project

protected internal
↓
Project OR Children

private protected
↓
Project + Children

public
↓
Everyone
```

A beginner sees:

```csharp
private int balance;
public void Deposit() { }
```

as syntax.

An experienced engineer sees:

> A carefully designed boundary that protects invariants, enforces encapsulation, reduces coupling, secures internal implementation details, and defines how software components collaborate safely and maintainably.

# C# CONSTRUCTORS

---

# Why Do Constructors Exist?

Before learning syntax like:

```csharp
Car car = new Car();
```

ask a deeper question:

> Why does C# need constructors at all?

Imagine creating a bank account object.

Without constructors:

```csharp
BankAccount account = new BankAccount();

account.AccountNumber = "12345";
account.Balance = 1000;
account.Owner = "Madan";
```

Now imagine a developer forgets:

```csharp
account.Owner = "Madan";
```

The object becomes:

```text
AccountNumber = "12345"
Balance = 1000
Owner = null
```

Problem:

```text
Invalid objects exist.
Bugs appear later.
Difficult debugging.
Unpredictable behavior.
```

Constructors exist because:

> Objects should start their life in a valid and usable state.

---

```text
The birth process of an object.
```

Example:

```text
Blueprint (Class)
↓
Construction Begins
↓
Constructor Prepares Object
↓
Object Becomes Ready
```

Without constructors:

```text
Newborn object
↓
Uninitialized
↓
Potentially broken
```

With constructors:

```text
Newborn object
↓
Initialized correctly
↓
Ready to use
```

# What Is a Constructor?

A constructor is:

> A special method that automatically runs when an object is created.

Its job is to:

```text
Initialize fields
Validate inputs
Prepare the object
Ensure a valid state
```

---

# Syntax

```csharp
class ClassName
{
    public ClassName()
    {
        // Initialization code
    }
}
```

---

# Example

```csharp
class Car
{
    public string Brand;

    public Car()
    {
        Brand = "Unknown";
    }
}
```

---

# Creating an Object

```csharp
Car car = new Car();

Console.WriteLine(car.Brand);
```

Output:

```text
Unknown
```

---

# Why Is It Called a Constructor?

Because:

> It constructs the object.

It takes an empty block of memory and transforms it into a usable object.

---

# Internal Working

Example:

```csharp
Car car = new Car();
```

Execution:

```text
new Car()
↓
Memory allocated on heap
↓
Fields receive default values
↓
Constructor executes
↓
Object initialized
↓
Reference returned
↓
Stored in variable
```

---

# Visual Representation

Before Constructor:

```text
Brand = null
Speed = 0
```

Constructor Runs:

```csharp
Brand = "Unknown";
```

After Constructor:

```text
Brand = Unknown
Speed = 0
```

---

# Constructors Are Special Methods

They look similar to methods.

But they are different.

---

# Constructor Rules

A constructor:

```text
Has the same name as the class.
Has no return type.
Runs automatically.
Can be overloaded.
```

---

# Example

Valid:

```csharp
class Student
{
    public Student()
    {
    }
}
```

---

Invalid:

```csharp
public void Student()
{
}
```

Because:

```text
Constructors do not have return types.
```

---

# Default Constructor

---

# What Is It?

A constructor with no parameters.

---

# Example

```csharp
class Person
{
    public Person()
    {
        Console.WriteLine("Person Created");
    }
}
```

---

# Creating Object

```csharp
Person p = new Person();
```

Output:

```text
Person Created
```

---

# Why Use Default Constructors?

Useful when:

```text
Objects need standard initialization.
Default values are sufficient.
```

---

# Compiler-Generated Default Constructor

Hidden behavior many beginners miss.

---

# Example

```csharp
class Car
{
    public string Brand;
}
```

No constructor written.

Still valid:

```csharp
Car car = new Car();
```

Why?

Because C# automatically provides:

```csharp
public Car()
{
}
```

---

# Important Rule

The compiler generates a default constructor ONLY IF:

```text
You don't define any constructor yourself.
```

---

# Example

```csharp
class Car
{
    public Car(string brand)
    {
    }
}
```

Now:

```csharp
new Car();
```

Error.

Because:

```text
Compiler stops generating the default constructor.
```

---

# Parameterized Constructors

---

# What Are They?

Constructors that accept inputs.

---

# Why Do They Exist?

Because objects often require information during creation.

Example:

A student without a name:

```text
Invalid.
```

---

# Example

```csharp
class Student
{
    public string Name;

    public Student(string name)
    {
        Name = name;
    }
}
```

---

# Creating Object

```csharp
Student s = new Student("Madan");

Console.WriteLine(s.Name);
```

Output:

```text
Madan
```

---

# Mental Model

Think:

```text
Object Birth Certificate
↓
Required Information Provided
↓
Object Becomes Valid
```

---

# Execution Flow

```text
new Student("Madan")
↓
Memory allocated
↓
Fields initialized to defaults
↓
name parameter receives "Madan"
↓
Constructor executes
↓
Name field assigned
↓
Object returned
```

---

# Multiple Parameters

Example:

```csharp
class Employee
{
    public string Name;
    public int Age;

    public Employee(string name,
                    int age)
    {
        Name = name;
        Age = age;
    }
```

---

# Creating Object

```csharp
Employee emp =
    new Employee("John", 30);
```

---

# Constructor Overloading

---

# What Is It?

Multiple constructors in the same class.

---

# Why?

Different ways to create objects.

---

# Example

```csharp
class Car
{
    public string Brand;

    public Car()
    {
        Brand = "Unknown";
    }

    public Car(string brand)
    {
        Brand = brand;
    }
}
```

---

# Creating Objects

```csharp
Car c1 = new Car();

Car c2 = new Car("Tesla");
```

---

# Output

```text
c1.Brand → Unknown

c2.Brand → Tesla
```

---

# Mental Model

Think:

```text
One Object
↓
Different Birth Processes
```

---

# Constructor Chaining

---

# What Is It?

Calling one constructor from another.

---

# Why Does It Exist?

Avoid duplicated initialization logic.

---

# Syntax

```csharp
: this(...)
```

---

# Example

```csharp
class Person
{
    public string Name;
    public int Age;

    public Person()
        : this("Unknown", 0)
    {
    }

    public Person(string name,
                  int age)
    {
        Name = name;
        Age = age;
    }
}
```

---

# Creating Objects

```csharp
Person p = new Person();
```

Execution:

```text
Default Constructor
↓
Calls Parameterized Constructor
↓
Initializes values
```

---

# Advantages

```text
Less duplication
Centralized initialization
Cleaner code
```

---

# The this Keyword in Constructors

---

# What Is It?

Refers to the current object.

---

# Example

```csharp
class Student
{
    public string Name;

    public Student(string Name)
    {
        this.Name = Name;
    }
}
```

---

# Why?

Without `this`:

```text
Parameter Name hides field Name.
```

---

# Mental Model

Think:

```text
this.Name
↓
Current object's field

Name
↓
Constructor parameter
```

---

# Readonly Fields and Constructors

---

# Why Important?

Readonly fields can only be assigned:

```text
During declaration
OR
Inside constructors.
```

---

# Example

```csharp
class Product
{
    public readonly int Id;

    public Product(int id)
    {
        Id = id;
    }
}
```

---

# Benefits

```text
Immutable state
Safer objects
```

---

# Hidden Behaviors and Edge Cases

---

# Fields Get Default Values First

Example:

```csharp
int → 0
bool → false
string → null
```

Before constructor runs.

---

# Constructors Cannot Be Inherited

Child classes do not inherit constructors.

---

# Constructors Cannot Return Values

Invalid:

```csharp
public int Car()
{
}
```

---

# Constructors Can Be Private

Used in:

```text
Singleton pattern
Factory methods
Controlled creation
```

---

# Static Constructors (Introduction)

---

# What Are They?

Initialize static members.

---

# Example

```csharp
class Config
{
    static Config()
    {
        Console.WriteLine("Static Initialized");
    }

```

---

# Characteristics

```text
No access modifiers.
No parameters.
Runs once.
Called automatically.
```

# Interview Perspective

## What is a constructor?

Answer:

A special method that automatically runs when an object is created to initialize it.

---

## Can constructors have return types?

Answer:

No.

Not even `void`.

---

## What is a default constructor?

Answer:

A constructor with no parameters.

---

## When does C# generate a default constructor?

Answer:

Only when no constructor is explicitly defined.

---

## What is constructor overloading?

Answer:

Having multiple constructors with different parameter lists.

---

## What is constructor chaining?

Answer:

Calling one constructor from another using `this()`.

---

## Why use constructors?

Answer:

To ensure objects begin life in a valid and usable state.

---

# Advantages of Constructors

```text
Object initialization
Improved consistency
Safer objects
Reduced bugs
Centralized setup logic
Supports immutability
Cleaner APIs
```

---

# Disadvantages

```text
Too many constructors can confuse users.
Poorly designed constructors become difficult to maintain.
Long parameter lists reduce readability.
```

---

# Best Practices Followed by Experienced Developers

- Ensure objects are valid after construction.
- Use parameterized constructors for required data.
- Use constructor chaining to avoid duplication.
- Keep constructors focused on initialization.
- Avoid heavy business logic inside constructors.
- Use readonly fields where appropriate.
- Limit the number of overloads.
- Prefer meaningful parameter names.

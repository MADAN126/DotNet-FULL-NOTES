# Why Do Break and Continue Exist?

Before learning:

```csharp
break;
continue;
```

ask a deeper question:

> Why aren't normal loop conditions enough?

Imagine you are searching for a person in a list of 10,000 employees.

Without `break`:

```text

â†“

```

Imagine processing student records.

You encounter one invalid record.

Without `continue`:

```text

OR
Write deeply nested if statements
```

Both approaches make code harder to understand.

These keywords exist because:

> Sometimes a loop needs to change its normal flow of execution.

Programs need the ability to say:

```text
"I'm done."
```

or

```text
"Skip this one."
```

# What Is continue?

`continue` skips the remainder of the current iteration and proceeds to the next iteration.

Example:

```csharp
continue;
```

---

# Why Does continue Exist?

Because sometimes:

> One item is invalid,
but the rest should still be processed.

Examples:

```text
Skip deleted users
Ignore empty lines
Ignore invalid input
Skip weekends
Process remaining records
```

---

# Syntax

```csharp

{
    if(condition)
    {
        continue;
    }

    // remaining code
}
```

# Break vs Continue

| Feature | break | continue |
|----------|--------|-----------|
| Stops Loop Completely | Yes | No |
| Skips Current Iteration | No | Yes |
| Continues Future Iterations | No | Yes |
| Used When | Goal achieved | Current item irrelevant |
| Affects | Entire loop | Current iteration |

# Mistake 5: Forgetting Remaining Code Is Skipped

# Hidden Behaviors and Edge Cases

---

# Break Immediately Transfers Control

Nothing after it executes inside the loop.

---

# Continue Still Executes Loop Updates in For Loops

Example:

```csharp

{
    continue;
}
```

The update section:

```text

```

---

# Continue Does NOT Execute Remaining Statements

Example:

```csharp

```

Skipped.

# Interview Perspective

## What does break do?

Answer:

It immediately terminates the nearest loop or switch statement.

---

## What does continue do?

Answer:

It skips the current iteration and proceeds with the next iteration.

---

## Difference between break and continue?

Answer:

`break` exits the loop entirely, while `continue` skips only the current iteration.

---

## Does break exit nested loops?

Answer:

No. It exits only the nearest enclosing loop.

---

## Can continue cause infinite loops?

Answer:

Yes, especially in `while` loops if the update logic is skipped.

---

## Why use break?

Answer:

To stop processing once further iterations become unnecessary.

---

# Best Practices Followed by Experienced Developers

- Use `break` when the objective has been achieved.
- Use `continue` to ignore invalid items while processing the rest.
- Avoid excessive use that obscures flow.
- Be extremely careful with `continue` inside `while` loops.
- Document complex loop exits clearly.
- Keep loop logic simple and readable.
- Think about performance when early termination is possible.
- Use these statements intentionally, not as shortcuts for poor design.

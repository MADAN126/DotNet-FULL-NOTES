# C# ARRAYS

---

# Why Do Arrays Exist?

Before learning syntax like:

```csharp
int[] numbers = { 10, 20, 30 };
```

ask a deeper question:

> Why do we even need arrays?

Imagine you need to store the marks of 5 students.

Without arrays:

```csharp
int mark1 = 85;
int mark2 = 90;
int mark3 = 78;
int mark4 = 92;
int mark5 = 88;
```

This works.

But what if:

```text
50 students?
500 students?
50,000 students?
```

Problems appear immediately:

```text
Huge amount of variables
Difficult to manage
Hard to process together
Impossible to scale
```

Now imagine:

```text
Display all products
Process all transactions
Calculate average marks
Search employees
Store game scores
```

Without arrays:

> Managing collections of related data would become a nightmare.

Arrays exist because:

> Programs frequently need to store multiple values of the same type under one logical name.

---

# Mental Model

Think of an array as:

```text
A row of lockers.
```

Example:

```text
Locker 0 → 85
Locker 1 → 90
Locker 2 → 78
Locker 3 → 92
Locker 4 → 88
```

Each locker:

```text
Stores one value
Has a numbered position
```

That position is called:

```text
Index
```

---

# What Is an Array?

An array is:

> A fixed-size collection of elements of the same data type stored under one variable name.

Example:

```csharp
int[] marks = { 85, 90, 78, 92, 88 };
```

---

# Breaking Down the Definition

Fixed-size:

```text
Size cannot change after creation.
```

Same data type:

```text
All elements must be int.
```

One variable:

```text
marks
```

Indexed access:

```text
marks[0]
marks[1]
```

---

# Why Fixed Size?

Arrays are designed for:

```text
Fast access
Efficient memory layout
Predictable structure
```

Trade-off:

```text
Speed
↓
Less flexibility
```

---

# Internal Working

Many beginners imagine:

```text
numbers
↓
Magic container
```

Reality:

An array is stored in contiguous memory.

Example:

```csharp
int[] numbers = {10, 20, 30};
```

Memory Concept:

```text
Address A     → 10
Address A+4   → 20
Address A+8   → 30
```

(assuming 4-byte integers)

---

# Why Contiguous Memory?

Benefits:

```text
Fast indexing
CPU cache efficiency
Simple address calculation
```

---

# Mental Model

```text

+
Index offset
=
Element location
```

# Why Zero-Based Indexing?

Historical reasons and efficiency.

Address formula:

```text
BaseAddress + Index × ElementSize
```

First element:

```text
Base + 0
```

Simple and fast.

# Array Length

---

# What Is Length?

The total number of elements.

Syntax:

```csharp
array.Length
```

Example:

```csharp
int[] numbers = {10,20,30};

Console.WriteLine(numbers.Length);
```

Output:

```text
3
```

---

# Why Is Length Important?

Avoid hardcoding.

Bad:

```csharp
for(int i=0;i<3;i++)
```

Good:

```csharp
for(int i=0;i<numbers.Length;i++)
```

---

# Iterating Arrays Using For Loop

Example:

```csharp
int[] numbers = {10,20,30};

for(int i=0;i<numbers.Length;i++)
{
    Console.WriteLine(numbers[i]);
}
```

Output:

```text
10
20
30
```

---

# Execution Flow

```text
i=0
↓
numbers[0]
↓
i=1
↓
numbers[1]
↓
i=2
↓
numbers[2]
↓
Stop
```

---

# Why Use For?

Because you need:

# When Should You Use Arrays?

Use arrays when:

```text
Size is known.

Performance matters.

Memory efficiency is important.

Data is fixed.
```

Examples:

```text
Months of year
Days of week
Pixel buffers
Sensor readings
Fixed configurations
```

---

# When Should You Avoid Arrays?

Avoid arrays when:

```text
Items frequently added.

Items frequently removed.

Collection size unknown.
```

Use:

```csharp
List<T>
```

instead.

# Hidden Behaviors and Edge Cases

---

# Arrays Know Their Length

Stored internally.

```csharp

```

---

# Length Is Read-Only

Invalid:

```csharp

```

Compilation Error.

---

# Arrays Cannot Grow

Need:

```text
New larger array
↓
Copy elements
```

---

# Foreach Variables Are Read-Only

Example:

```csharp

{
    n++;
}
```

Compilation Error.

# Interview Perspective

## What is an array?

Answer:

An array is a fixed-size collection of elements of the same type stored in contiguous memory and accessed using indexes.

---

## Why are arrays zero-indexed?

Answer:

Because address calculation becomes simpler and more efficient.

---

## Are arrays value types or reference types?

Answer:

Arrays are reference types.

---

## Difference between array and List<T>?

Answer:

Arrays have fixed size, while List<T> can dynamically grow and shrink.

---

## What happens if you access an invalid index?

Answer:

An `IndexOutOfRangeException` is thrown.

---

## Why use foreach with arrays?

Answer:

It provides a cleaner and safer way to iterate when indexes are unnecessary.

---

# Best Practices Followed by Experienced Developers

- Use arrays when size is fixed.
- Prefer `Length` over hardcoded values.
- Validate indexes before access.
- Use `foreach` for read-only traversal.
- Use `for` when indexes are needed.
- Remember arrays are reference types.
- Clone arrays when independent copies are required.
- Choose `List<T>` when collections must grow.

```text
Need Multiple Values
↓
Same Data Type
↓
Fixed Number of Elements
↓
Allocate Contiguous Memory
```

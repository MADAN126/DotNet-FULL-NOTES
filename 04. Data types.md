# C# DATA TYPES

---

# Why Do Data Types Exist?

Before learning C# data types, ask a deeper question:

> Why can't computers just store "data" without types?

Imagine a warehouse.

You receive:

- Books
- Milk cartons
- Furniture
- Chemicals
- Glass items

Would you store all of them in the same box?

Of course not.

Each item has:

- Different sizes
- Different handling requirements
- Different behaviors
- Different safety concerns

Computers face the same problem.

Different kinds of data require different treatment.

Examples:

```text
25          → Age
99.99       → Price
true        → IsLoggedIn
'A'         → Grade
"John"      → Name
```

They may all look like "data" to humans.

To computers, they are fundamentally different.

Data types exist to tell the compiler:

> "This variable will store a specific kind of information."

---

# What Are Data Types?

A data type defines:

- What kind of value can be stored
- How much memory is required
- What operations are allowed
- How the CLR handles the value internally

Example:

```csharp
int age = 25;
```

This tells C#:

```text
Variable Name : age
Type          : int
Value         : 25
```

---

# Mental Model

Think of data types as containers designed for specific purposes.

Example:

```text
Coffee Cup       → Coffee
Petrol Tank      → Fuel
Pen Drive        → Files
Water Bottle     → Water
```

Similarly:

```text
int      → Whole numbers
double   → Decimal numbers
bool     → True/False
string   → Text
char     → Single characters
```

---

# Why Are Data Types Important?

Without data types:

The compiler wouldn't know:

```text
How much memory to allocate?
How should arithmetic work?
Can these values be compared?
Can this operation be allowed?
```

Example:

```csharp
"John" / true
```

What should this even mean?

Data types prevent nonsense.

---

# What Problems Do Data Types Solve?

---

## Problem 1: Memory Management

Different values require different amounts of memory.

Example:

```text
true      → Very small
1,000,000 → Larger
Long text → Much larger
```

---

## Problem 2: Type Safety

Incorrect:

```csharp
int age = "Twenty";
```

Compiler Error.

---

## Problem 3: Performance

Using an appropriate type improves efficiency.

Example:

```csharp
bool isActive;
```

is more efficient than:

```csharp
string isActive = "true";
```

---

## Problem 4: Readability

Example:

```csharp
decimal accountBalance;
```

Immediately communicates intent.

This distinction becomes extremely important later.

---

## What Is It?

## Hidden Behavior

Floating-point numbers cannot precisely represent all decimals.

Example:


```text

```

---

---

## What Is It?

A high-precision decimal type.

Example:


```text

```

## Memory Size

```text

128 bits
```

## Why Does It Exist?

Money requires precision.

Example:

Incorrect:


Financial systems cannot tolerate this.

---

## Use Cases

```csharp

decimal taxAmount;
```

---

## Advantages

- High precision
- Accurate for financial calculations

---

## Disadvantages

- Slightly slower
- Uses more memory

# string

---

## What Is It?

Stores sequences of characters.

Example:

```csharp

```

---

## Why Does It Exist?

Text is everywhere.

Examples:

```text

Addresses
Emails
Messages
Passwords
```

---

## Internal Working

Strings are objects.

Memory:

```text
Variable
↓
Reference
↓

```

---

## Hidden Behavior

Strings are immutable.

Example:

```csharp
string name = "John";

name = "Jack";
```

It appears modified.

Reality:

```text
Old String → "John"
New String → "Jack"
Reference Updated
```

# var (Implicit Typing)

---

# What Is It?

Allows compiler to infer the type.

Example:

```csharp
var age = 25;
```

Compiler interprets:

```csharp
int age = 25;
```

---

# Why Does It Exist?

Reduces verbosity.

---

# Common Misconception

Misconception:

> var means dynamic typing.

Reality:

The type is still determined at compile time.

Example:

```csharp
var age = 25;

age = "John";
```

Compiler Error.

---

# Comparison: var vs Explicit Types

| Feature | var | Explicit |
|----------|-----|----------|
| Type Inference | Yes | No |
| Compile-Time Safety | Yes | Yes |
| Readability | Context Dependent | High |
| Dynamic | No | No |

---

# Type Conversion

Sometimes data moves between types.

Example:

```csharp
int age = 25;

double result = age;
```

Allowed.

---

# Implicit Conversion

Automatic.

Example:

```csharp
int number = 10;
double value = number;
```

---

# Explicit Conversion

Manual.

Example:

```csharp
double number = 10.8;

int result = (int)number;
```

Output:

```text
10
```

Fraction lost.

# Hidden Behaviors and Edge Cases

---

## Floating-Point Precision Errors

Example:

```csharp
0.1 + 0.2 != 0.3
```

Sometimes true with doubles.

---

## Strings Are Immutable

Modifications create new objects.

---

## Boolean Values Are Strict

Only:

```text
true
false
```

Unlike some languages:

```text
0 → false
1 → true
```

C# does not allow this.

---

## Overflow Can Occur

Example:

```csharp
int max = int.MaxValue;

max++;
```

May wrap around in unchecked contexts.

# Interview Perspective

## What is a data type?

Answer:

A data type specifies the kind of value a variable can store, how much memory it uses, and what operations are permitted.

---

## Difference between value types and reference types?

Answer:

Value types store actual data directly, while reference types store references pointing to objects.

---

## Why use decimal instead of double for money?

Answer:

Because decimal provides higher precision and avoids floating-point rounding issues.

---

## What is the difference between char and string?

Answer:

char stores a single Unicode character using single quotes, while string stores multiple characters using double quotes.

---

## Is var dynamically typed?

Answer:

No. The compiler determines its type during compilation, and it remains strongly typed.

---

## What is implicit conversion?

Answer:

Automatic conversion from a smaller compatible type to a larger type without data loss.

---

# Best Practices Followed by Experienced Developers

- Use the most appropriate type for the problem.
- Prefer decimal for financial calculations.
- Use bool for decisions instead of strings like "Yes"/"No".
- Avoid object unless flexibility is truly needed.
- Use meaningful variable names.
- Use var when the type is obvious from the right-hand side.
- Be aware of overflow and precision limitations.
- Understand value vs reference behavior early.

---

# Final Mental Model

Think of data types as contracts between you, the compiler, and the CLR.

```text
Real-World Information
↓
Choose Appropriate Data Type
↓
Compiler Enforces Rules
↓
CLR Allocates Proper Memory
↓
Operations Become Safe and Predictable
↓
Program Behaves Correctly
```

A beginner sees:

```csharp
decimal salary = 50000m;
```

as syntax.

An experienced engineer sees:

> A deliberate choice balancing precision, memory usage, performance, type safety, and domain requirements to accurately model real-world financial data.

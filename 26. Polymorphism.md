# C# INHERITANCE

---

# Why Does Inheritance Exist?

Before learning syntax like:

```csharp
class Dog : Animal
{
}
```

ask a deeper question:

> Why would one class inherit from another?

Imagine building a zoo management system.

Without inheritance:

```csharp
class Dog
{
    public string Name;
    public int Age;

    public void Eat()
    {
    }

    public void Sleep()
    {
    }
}

class Cat
{
    public string Name;
    public int Age;

    public void Eat()
    {
    }

    public void Sleep()
    {
    }
}

class Lion
{
    public string Name;
    public int Age;

    public void Eat()
    {
    }

    public void Sleep()
    {
    }
}
```

Problems:

```text
Code duplication.
Hard maintenance.
Inconsistent behavior.
Bugs fixed in one place but forgotten elsewhere.
```

If every animal:

```text
Eats
Sleeps
Has a name
Has an age
```

why repeat it?

Inheritance exists because:

> Classes often share common characteristics, and software needs a way to reuse and extend existing behavior.

---

# What Is Inheritance?

Inheritance means:

> A class can acquire the members of another class.

It allows us to build:

```text
Specialized classes
↓
using
↓
General classes
```

---

# Mental Model

Think of a family tree.

```text
Animal
│
├── Dog
├── Cat
└── Lion
```

Children inherit traits from parents.

Example:

```text
Eye color
Hair color
Height tendencies
```

Similarly in software:

```text
Fields
Methods
Properties
Behavior
```

can be inherited.

---

# The Core Philosophy

Instead of saying:

```text
Dog has Name.
Dog has Eat().
Dog has Sleep().
```

we say:

```text
A Dog IS AN Animal.
```

This is called the:

> IS-A Relationship.

---

# Mental Model

Ask:

```text
Is Dog an Animal?
```

Yes.

Inheritance fits.

Ask:

```text
Is Engine a Car?
```

No.

Inheritance does NOT fit.

That's composition.

# What Does NOT Get Inherited?

Very important.

---

# Constructors

Constructors are NOT inherited.

Example:

```csharp
class Animal
{
    public Animal()
    {
    }
}
```

Dog does NOT inherit:

```csharp
Animal()
```

---

# Private Members

Private members remain inside the parent.

Example:

```csharp
class Animal
{
    private int age;
}
```

Child:

```csharp
class Dog : Animal
{
}
```

Cannot access:

```csharp
age
```

---

# Why?

Because:

> Private means only the declaring class can access it.

---

# Access Modifier Behavior

| Modifier | Inherited? | Directly Accessible? |
|------------|------------|-----------------------|
| public | Yes | Yes |
| protected | Yes | Yes |
| private | No | No |
| internal | Depends on assembly | Usually |
| protected internal | Yes | Yes |
| private protected | Limited | Limited |

---

# protected and Inheritance

One major reason protected exists.

Example:

```csharp
class Animal
{
    protected string Name;
}

class Dog : Animal
{
    public void SetName()
    {
        Name = "Rocky";
    }
}
```

Valid.

Outside:

```csharp
dog.Name = "Rocky";
```

Error.

# Constructor Execution Order

One of the most important concepts.

Example:

```csharp
class Animal
{
    public Animal()
    {
        Console.WriteLine("Animal Constructor");
    }
}

class Dog : Animal
{
    public Dog()
    {
        Console.WriteLine("Dog Constructor");
    }
}
```

---

# Usage

```csharp
Dog dog = new Dog();
```

Output:

```text
Animal Constructor
Dog Constructor
```

Parent initializes first
↓

# Base Keyword

---

# What Is It?

Used to access members of the base class.

---

# Syntax

```csharp
base.Member
```

---

# Example

```csharp
class Animal
{
    protected string Name = "Animal";
}

class Dog : Animal
{
    public void Display()
    {
        Console.WriteLine(base.Name);
    }
}
```

Output:

```text
Animal
```

---

# Calling Parent Constructor

Example:

```csharp
class Animal
{
    public Animal(string name)
    {
        Console.WriteLine(name);
    }
}

class Dog : Animal
{
    public Dog()
        : base("Rocky")
    {
    }
}
```

---

# Execution

```text
Dog()
↓
base("Rocky")
↓
Animal Constructor Runs
↓
Dog Constructor Runs
```

# Single Inheritance in C#

C# supports:

```text
One Parent
Multiple Children
```

---

# Valid

```text
Animal
↓
Dog
```

---

# Invalid

```text
Animal
   ↘
    Dog
   ↗
Pet
```

Example:

```csharp
class Dog : Animal, Pet
{
}
```

Compilation Error.

---

# Why No Multiple Inheritance?

Because it creates problems like:

```text
Ambiguity.
Diamond Problem.
Complexity.
```

---

# The Diamond Problem

Imagine:

```text
Animal
 /    \
Pet  Mammal
 \    /
  Dog
```

Suppose both parents have:

```csharp
Eat()
```

Question:

```text
Which Eat() should Dog use?
```

Confusing.

C# avoids this by:

> Allowing only single class inheritance.

# Inheritance vs Composition

One of the biggest design topics.

---

# Inheritance

Relationship:

```text
IS-A
```

Example:

```text
Dog IS AN Animal.
```

---

# Composition

Relationship:

```text
HAS-A
```

Example:

```text
Car HAS AN Engine.
```

---

# Comparison Table

| Feature | Inheritance | Composition |
|-----------|------------|-------------|
| Relationship | IS-A | HAS-A |
| Reuse | Through hierarchy | Through objects |
| Coupling | Tight | Loose |
| Flexibility | Lower | Higher |
| Real-World Example | Dog → Animal | Car → Engine |

---

# Advantages of Inheritance

---

# 1. Code Reuse

Common functionality written once.

---

# 2. Easier Maintenance

Fix parent.

Children benefit.

---

# 3. Logical Hierarchies

Natural modeling.

---

# 4. Extensibility

Derived classes add behavior.

---

# 5. Supports Polymorphism

Foundation for runtime polymorphism.

---

# Disadvantages of Inheritance

---

# 1. Tight Coupling

Children depend heavily on parents.

---

# 2. Fragile Base Class Problem

Changes in parent can break children.

---

# 3. Reduced Flexibility

Deep hierarchies become difficult.

---

# 4. Overuse Leads to Complexity

Large inheritance trees are hard to understand.

# Interview Perspective

## What is inheritance?

Answer:

Inheritance is an OOP feature that allows one class to acquire members from another class.

---

## What relationship does inheritance represent?

Answer:

An IS-A relationship.

---

## What is a base class?

Answer:

The parent class being inherited from.

---

## What is a derived class?

Answer:

The child class inheriting from the parent.

---

## Are constructors inherited?

Answer:

No.

However, the parent constructor executes before the child constructor.

---

## Does C# support multiple inheritance?

Answer:

No.

C# supports single class inheritance.

---

## Difference between inheritance and composition?

Answer:

Inheritance models IS-A relationships, while composition models HAS-A relationships.

---

# Best Practices Followed by Experienced Developers

- Use inheritance only for true IS-A relationships.
- Prefer composition when flexibility is needed.
- Keep inheritance hierarchies shallow.
- Avoid inheriting solely for code reuse.
- Use protected carefully.
- Design stable base classes.
- Favor extensibility over rigidity.
- Remember that inheritance introduces coupling.

# C# ABSTRACTION

---

# Why Does Abstraction Exist?

Before learning keywords like:

```csharp
abstract
abstract class
abstract method
override
interface
```

ask a deeper question:

> Why should software hide details and expose only essential behavior?

Imagine driving a car.

You interact with:

```text
Steering Wheel
Brake Pedal
Accelerator
Gear Lever
```

You DO NOT interact with:

```text
Fuel injection timing
Engine combustion cycles
Transmission algorithms
Brake pressure calculations
ECU firmware
```

The car hides complexity.

You only see:

```text
What you need to use.
```

Now imagine software without abstraction.

To send an email, you might need to know:

```text
SMTP packet structures
Socket communication
TLS handshakes
Authentication internals
Retry mechanisms
```

Programming would become overwhelming.

Abstraction exists because:

> Users of a component should focus on WHAT it does, not HOW it does it.

---

# What Is Abstraction?

Abstraction means:

> Hiding implementation details while exposing only the essential features.

---

# Simple Definition

Abstraction answers:

```text
WHAT should this object do?
```

It hides:

```text
HOW does it do it?
```

---

# Mental Model

Think of a television remote.

You press:

```text
Power
Volume Up
Channel Down
```

You don't know:

```text
Infrared signals
Circuit design
Processor instructions
Electrical pathways
```

The remote abstracts complexity.

---

# The Core Philosophy

Instead of exposing:

```text
Every implementation detail.
```

Expose:

```text

```

Example:

```text

```

Not:

```text

```

```text

```

```text

```

# Abstraction

Focuses on:

```text
Hiding complexity.

```

Question:

```text

```

# Abstract Classes

---

# What Is an Abstract Class?

An abstract class is:

> A class that cannot be instantiated directly and is designed to be inherited.

---

# Why Do Abstract Classes Exist?

Sometimes a concept is too general to create objects.

Example:

```text
Animal
Vehicle
Payment
Shape
Employee
```

Question:

Can you create a generic:

```text
Animal?
```

Not really.

You create:

```text
Dog
Cat
Lion
```

```csharp
abstract class Animal
{
}
```

# Abstract Methods

---

# What Are They?

Methods without implementation.

---

# Why?

Because the parent knows:

```text
WHAT should happen.
```

but not:

```text
HOW it should happen.
```

---

# Example

```csharp
abstract class Animal
{
    public abstract void MakeSound();
}
```

---

# Interpretation

Animal says:

```text
Every animal makes sound.

I don't know which sound.
```

---

# Derived Classes

```csharp
class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Bark");
    }
}

class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Meow");
    }
}
```

---

# Usage

```csharp
Animal dog = new Dog();

dog.MakeSound();
```

Output:

```text
Bark
```

---

# Execution Flow

```text
Animal Reference
↓
Dog Object
↓
Dog's MakeSound()
↓
Bark
```

---

# Rules of Abstract Methods

Abstract methods:

```text
Have no body.
Must be inside abstract classes.
Must be overridden by concrete children.
Cannot be private.
Cannot be static.
```

---

# Example

Valid:

```csharp
public abstract void Run();
```

---

Invalid:

```csharp
public abstract void Run()
{
}
```

---

# Abstract Classes Can Have Implementations

Many beginners misunderstand this.

Abstract classes can contain:

```text
Fields
Properties
Methods
Constructors
Concrete methods
Abstract methods
```

---

# Example

```csharp
abstract class Animal
{
    public void Eat()
    {
        Console.WriteLine("Eating...");
    }

    public abstract void MakeSound();
}
```

---

# Usage

```csharp

dog.MakeSound();
```

Output:

```text

Bark
```

---

# Why?

Abstract classes provide:

```text
Shared behavior
+
Required behavior
```

# Interfaces and Abstraction

Interfaces represent pure contracts.

---

# What Is an Interface?

An interface is:

> A contract specifying what must be done, without specifying how.

---

# Example

```csharp
interface IPayment
{
    void Pay(decimal amount);
}
```

---

# Implementations

```csharp
class CreditCardPayment : IPayment
{
    public void Pay(decimal amount)
    {
        Console.WriteLine("Credit Card");
    }
}

class UPIPayment : IPayment
{
    public void Pay(decimal amount)
    {
        Console.WriteLine("UPI");
    }
}
```

---

# Usage

```csharp
IPayment payment = new UPIPayment();

payment.Pay(500);
```

Output:

```text
UPI
```

---

# Mental Model

Think:

```text
Interface

↓

Promise

↓

Implementations Fulfill Promise
```

# Abstract Class vs Interface

One of the most common interview questions.

---

# Comparison Table

| Feature | Abstract Class | Interface |
|-----------|---------------|------------|
| Can Have Fields | Yes | No |
| Constructors | Yes | No |
| Method Implementations | Yes | Traditionally No* |
| Abstract Methods | Yes | Yes |
| Multiple Inheritance | No | Yes |
| IS-A Relationship | Yes | Capability Contract |
| Instantiation | No | No |

*Modern C# allows default interface implementations.

---

# When To Use Abstract Classes

Use abstract classes when:

```text
Classes share state.
Classes share implementation.
Strong inheritance relationship exists.
You want default behavior.
```

---

# Example

```text
Animal
Vehicle
Employee
Document
```

---

# When To Use Interfaces

Use interfaces when:

```text
You need capabilities.
Unrelated classes share behavior.
Multiple implementations exist.
Loose coupling is desired.
```

---

# Example

```text
ILogger
IDisposable
IComparable
IPayment
```

# Advantages of Abstraction

---

# 1. Reduces Complexity

Users focus on essentials.

---

# 2. Improves Maintainability

Internal changes don't affect consumers.

---

# 3. Encourages Loose Coupling

Depend on contracts.

---

# 4. Supports Polymorphism

Different implementations.

Same interface.

---

# 5. Enhances Extensibility

New implementations added easily.

---

# Disadvantages

---

# 1. More Initial Design Effort

Requires planning.

---

# 2. Over-Abstraction

Too many abstractions confuse developers.

---

# 3. Harder For Beginners

Understanding indirect behavior takes time.

# Interview Perspective

## What is abstraction?

Answer:

Abstraction is hiding implementation details while exposing essential behavior.

---

## How is abstraction achieved in C#?

Answer:

Using abstract classes and interfaces.

---

## Can abstract classes be instantiated?

Answer:

No.

---

## Can abstract classes contain concrete methods?

Answer:

Yes.

---

## Difference between abstract classes and interfaces?

Answer:

Abstract classes support shared state and implementation, while interfaces define contracts and support multiple implementations.

---

## Why is abstraction important?

Answer:

It reduces complexity, improves maintainability, encourages loose coupling, and supports extensibility.

---

# Best Practices Followed by Experienced Developers

- Expose only essential operations.
- Program against abstractions rather than concrete classes.
- Use abstract classes for shared behavior.
- Use interfaces for capabilities and contracts.
- Avoid premature abstraction.
- Keep abstractions meaningful.
- Don't leak implementation details.
- Prefer simple APIs.

# C# METHOD PARAMETERS, DEFAULT PARAMETER VALUES, RETURN VALUES & NAMED ARGUMENTS

---

# Why Do These Concepts Exist?

Before learning syntax like:

```csharp
Add(10, 20);
Greet("John");
CalculateTax(price);
```

ask a deeper question:

> Why do methods need inputs and outputs at all?

Imagine a world where methods couldn't receive values.

You would have to create methods like this:

```csharp
PrintJohn();
PrintAlex();
PrintMary();

AddTwoNumbers();
AddThreeNumbers();
AddTenNumbers();
```

Problems:

```text
Massive duplication
Rigid design
Difficult maintenance
Poor reusability
Code explosion
```

Now imagine methods that cannot return values.

Example:

```text
Calculate total bill
Find maximum salary
Validate password
Calculate discount
Convert temperature
```

Without return values:

```text
Methods become useless calculators.
You can perform work,
but you cannot obtain results.
```

These features exist because:

> Methods should behave like reusable machines that accept inputs, perform work, and optionally produce outputs.


# What Are Parameters?

Parameters are:

> Variables defined by a method that receive values from the caller.

---

# Why Do Parameters Exist?

Without parameters:

Methods become fixed.

Example:

Bad:

```csharp
static void GreetJohn()
{
    Console.WriteLine("Hello John");
}
```

Want to greet Alex?

Need another method.

Bad:

```csharp
static void GreetAlex()
{
    Console.WriteLine("Hello Alex");
}
```

This does not scale.

---

# Parameters Solve This

Example:

```csharp
static void Greet(string name)
{
    Console.WriteLine("Hello " + name);
}
```

Now:

```csharp
Greet("John");
Greet("Alex");
Greet("Mary");
```

Output:

```text
Hello John
Hello Alex
Hello Mary
```

---

# Mental Model

Think:

```text
Method = Machine

Parameters = Raw Materials
```

Example:

```text
Bread + Cheese
↓
Sandwich Machine
↓
Sandwich
```

---

# Multiple Parameters

Methods can receive multiple inputs.

Example:

```csharp
static void Display(string name, int age)
{
    Console.WriteLine(name);
    Console.WriteLine(age);
}
```

Calling:

```csharp
Display("John", 25);
```

Output:

```text
John
25
```

---

# Execution Flow

```text
"John" → name
25 → age
↓
Method Body Executes
↓
Method Ends
```

# Parameter Order Matters

Example:

```csharp
Display("John", 25);
```

Correct.

---

Incorrect:

```csharp
Display(25, "John");
```

Compilation Error.

---

# Why?

Because C# matches arguments by position.

# Parameters vs Arguments

One of the most common interview questions.

---

# Parameters

Defined by method.

Example:

```csharp
static void Greet(string name)
```

Here:

```text
name
```

is a parameter.

---

# Arguments

Actual values passed.

Example:

```csharp
Greet("John");
```

Here:

```text
"John"
```

is an argument.

---

# Comparison

| Feature | Parameter | Argument |
|----------|-----------|-----------|
| Defined In | Method declaration | Method call |
| Purpose | Receives values | Supplies values |
| Example | `string name` | `"John"` |

---

# Default Parameter Values

---

# What Are Default Parameters?

They allow methods to use predefined values when callers don't provide arguments.

---

# Why Do They Exist?

Without defaults:

Example:

```csharp
SendEmail("admin", false);
SendEmail("john", false);
SendEmail("alex", false);
```

Suppose:

```text
false
```

is almost always used.

This becomes repetitive.

---

# Default Parameters Solve This

Example:

```csharp
static void SendEmail(string user,
                      bool urgent = false)
{
    Console.WriteLine(user);
    Console.WriteLine(urgent);
}
```

---

# Calling With Defaults

```csharp
SendEmail("John");
```

Output:

```text
John
False
```

---

# Calling With Custom Values

```csharp
SendEmail("John", true);
```

Output:

```text
John
True
```

---

# Mental Model

Think:

```text
"If you don't tell me,
I'll assume this value."
```

---

# Execution Flow

Call:

```csharp
SendEmail("John");
```

Compiler sees:

```text
Missing urgent argument
↓
Uses default false
↓
Method executes
```

---

# Rules for Default Parameters

Default parameters must come after required parameters.

Correct:

```csharp
void Test(int x,
          int y = 10)
{
}
```

---

Incorrect:

```csharp
void Test(int x = 10,
          int y)
{
}
```

Compilation Error.

---

# Why?

Otherwise the compiler cannot determine argument positions.

---

# Real-World Usage

Examples:

```text
Logging levels
Email notifications
Pagination limits
Timeout values
Formatting options
Feature toggles
```

# Return Values

---

# What Is a Return Value?

A return value is:

> The result sent back from a method to the caller.

---

# Why Do Return Values Exist?

Methods often calculate information.

Examples:

```text
Add numbers
Calculate taxes
Validate users
Determine eligibility
Find maximum values
```

Without returns:

```text
Results disappear.
```

---

# Mental Model

Think:

```text
Input
↓
Machine
↓
Output
```

---

# Syntax

```csharp
static returnType MethodName(...)
{
    return value;
}
```

---

# Example

```csharp
static int Add(int a, int b)
{
    return a + b;
}
```

Calling:

```csharp
int result = Add(5, 3);

Console.WriteLine(result);
```

Output:

```text
8
```

---

# Step-by-Step Execution

```text
Add(5,3)
↓
a = 5
b = 3
↓
Calculate 8
↓
return 8
↓
Caller receives value
↓
Store in result
```

---

# Why Is Return Better Than Printing?

Bad:

```csharp
static void Add(int a, int b)
{
    Console.WriteLine(a + b);
}
```

Problem:

```text
Cannot reuse result.
Cannot perform further calculations.
Harder to test.
```

---

Good:

```csharp
static int Add(int a, int b)
{
    return a + b;
}
```

Now:

```csharp
int total = Add(5,3);

Console.WriteLine(total);

if(total > 5)
{
}
```

---

# The return Statement

The `return` keyword does two things.

---

# Function 1

Returns a value.

Example:

```csharp
return 10;
```

---

# Function 2

Immediately exits the method.

Example:

```csharp
return 10;

Console.WriteLine("Never runs");
```

---

# Hidden Behavior

Code after return:

```text
Unreachable.
```

---

# void vs Return Values

| Feature | void | Returning Method |
|-----------|------|-------------------|
| Produces Value | No | Yes |
| Uses return value | No | Yes |
| Purpose | Action | Calculation |
| Example | Print() | Add() |

# Named Arguments

---

# What Are Named Arguments?

Named arguments allow arguments to be matched by parameter names rather than positions.

---

# Why Do They Exist?

Suppose:

```csharp
CreateUser("John",
           true,
           false,
           true);
```

Question:

```text
What does each boolean mean?
```

Hard to understand.

---

# Named Arguments Solve This

Example:

```csharp
CreateUser(
    name: "John",
    isAdmin: true,
    emailVerified: false,
    active: true
);
```

Much clearer.

---

# Syntax

Example:

```csharp
Display(
    age: 25,
    name: "John"
);
```

Method:

```csharp
static void Display(string name,
                    int age)
{
}
```

---

# Why Does This Work?

Because C# matches:

```text
Parameter names
instead of
positions.
```

---

# Benefits

```text
Improved readability
Self-documenting code
Flexible ordering
Fewer mistakes
```

---

# Execution Flow

Call:

```csharp
Display(age:25,
        name:"John");
```

Compiler sees:

```text
age → age parameter
name → name parameter
↓
Reorders internally
↓
Method executes normally
```

---

# Combining Named Arguments with Defaults

Example:

```csharp
static void Register(
    string username,
    bool admin = false,
    bool active = true)
{
}
```

Calling:

```csharp
Register(
    username: "Madan",
    active: false
);
```

Equivalent to:

```csharp
Register(
    "Madan",
    false,
    false
);
```

```csharp
ConnectDatabase(

```

Much easier to understand than:


# Hidden Behaviors and Edge Cases

---

# Parameters Are Local Variables

Example:

```csharp
static void Test(int x)
{
}
```

`x` exists only during execution.

---

# Default Values Are Compile-Time Constants

Valid:

```csharp
int x = 5
bool flag = false
string name = "Guest"
```

---

Invalid:

```csharp
DateTime.Now
```

Cannot be used as default values.

---

# Named Arguments Improve Readability

But excessive usage can make calls verbose.

---

# Return Ends Execution Immediately

Nothing after return executes.

---

# Methods Must Return on Every Path

Example:

Incorrect:

```csharp
static int Test(bool flag)
{
    if(flag)
    {
        return 1;
    }
}
```

Compilation Error.

---

Correct:

```csharp
static int Test(bool flag)
{
    if(flag)
    {
        return 1;
    }

    return 0;
}
```

# Interview Perspective

## What is a parameter?

Answer:

A variable defined by a method that receives input values.

---

## What is an argument?

Answer:

The actual value passed to a method.

---

## What are default parameters?

Answer:

Parameters with predefined values used when arguments are omitted.

---

## Why use default parameters?

Answer:

To reduce repetitive arguments and simplify method calls.

---

## What is a return value?

Answer:

A value sent back from a method to its caller.

---

## What does return do?

Answer:

It returns a value and immediately exits the method.

---

## What are named arguments?

Answer:

Arguments specified using parameter names rather than positions.

---

## Advantages of named arguments?

Answer:

Improved readability, flexibility, and reduced mistakes.

---

# Best Practices Followed by Experienced Developers

- Keep parameter lists short.
- Use meaningful parameter names.
- Prefer returning values over printing results.
- Use default parameters for commonly repeated values.
- Use named arguments when they improve clarity.
- Ensure every execution path returns a value.
- Avoid methods with too many boolean parameters.
- Design methods as small, focused units of behavior.

---

# Final Mental Model

Think of methods as service machines.

```text
Need Information?
↓
Provide Parameters

Need Optional Behavior?
↓
Use Default Parameters

Need Readable Calls?
↓
Use Named Arguments

Need Results?
↓
Return Values

Method Executes
↓
Caller Continues
```

A beginner sees:

```csharp
int total = Add(5, 3);
```

as syntax.

An experienced engineer sees:

> A contract between caller and method where inputs are supplied through parameters, defaults simplify common scenarios, named arguments improve intent, and return values allow methods to participate in larger workflows by producing reusable results.

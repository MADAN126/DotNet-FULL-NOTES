# C# OPERATORS

---

# Why Do Operators Exist?

Before learning symbols like:

```csharp
+
-
*
/
==
&&
=
```

ask a deeper question:

> Why do programming languages need operators at all?

Imagine trying to describe every operation using long sentences.

Instead of:

```csharp
total = price + tax;
```

You would have to write:

```text
Take the value stored in price,
add the value stored in tax,
and store the result in total.
```

For every calculation.

For every comparison.

For every decision.

Programs would become extremely verbose.

Operators exist because:

> They provide a compact language for expressing computations, comparisons, assignments, and logical decisions.

They are the verbs of programming.

Variables are nouns.

Operators describe what happens to those nouns.

Example:

```text

Output
```

Example:

```text

```

Or:

```text

```

Operators transform values.

---

# What Is an Operator?

An operator is a symbol that instructs the compiler to perform a specific operation on one or more operands.

Example:

```csharp
5 + 3
```

Here:

```text
5 and 3 → Operands
+       → Operator
```

Result:

```text
8
```

---

# Why Do Operators Matter?

Without operators:

Programs could not:

- Perform calculations
- Make decisions
- Compare values
- Assign data
- Combine conditions
- Modify variables efficiently

Almost every line of code uses operators.

| false | false | false |

# Hidden Behavior: Short-Circuit Evaluation

One of the most important concepts.

---

# AND (&&)

Example:

```csharp
false && SomethingExpensive()
```

Execution:

```text
false found
↓
Result already false
↓
Second condition skipped
```

---

# OR (||)

Example:

```csharp
true || SomethingExpensive()
```

Execution:

```text
true found
↓
Result already true
↓
Second condition skipped
```

---

# Why Does This Exist?

Improves:

- Performance
- Safety

---

# Example

Safe:

```csharp
if (name != null && name.Length > 0)
{
}
```

If name is null:

```text
Second condition never executes.
```

Avoids exceptions.

---

# Prefix vs Postfix

---

# Prefix

Example:

```csharp
++x;
```

Meaning:

```text
Increment first
Then use value
```

---

# Postfix

Example:

```csharp
x++;
```

Meaning:

```text
Use current value
Then increment
```

---

# Example

```csharp
int x = 5;

Console.WriteLine(x++);
```

Output:

```text
5
```

Afterward:

```text
x = 6
```

---

Example:

```csharp
int x = 5;

Console.WriteLine(++x);
```

Output:

```text
6
```

# Operator Precedence

---

# Why Does It Exist?

Consider:

```csharp
2 + 3 * 4
```

Should it mean:

```text
(2 + 3) × 4 = 20
```

or:

```text
2 + (3 × 4) = 14
```

Rules are needed.

---

# Common Precedence Order

```text
()
↓
Unary (++ -- !)
↓
* / %
↓
+ -
↓
Comparison
↓
Logical AND
↓
Logical OR
↓
Assignment
```

---

# Example

```csharp
int result = 2 + 3 * 4;
```

Execution:

```text
3 × 4 = 12
↓

2 + 12 = 14
```

---

# Use Parentheses for Clarity

Good:

```csharp
int result = (2 + 3) * 4;
```

Output:

```text
20
```

# Interview Perspective

## What is an operator?

Answer:

An operator is a symbol that performs an operation on one or more operands.

---

## Difference between = and ==?

Answer:

`=` assigns values, while `==` compares values.

---

## What is short-circuit evaluation?

Answer:

Logical operators `&&` and `||` stop evaluating once the final result is already known.

---

## What is integer division?

Answer:

Division between integers discards the fractional part.

Example:

```csharp
5 / 2
```

returns:

```text
2
```

---

## Difference between x++ and ++x?

Answer:

`x++` uses the current value before incrementing, while `++x` increments first and then uses the updated value.

---

## Why use modulus (%)?

Answer:

It returns the remainder and is commonly used for tasks like checking even or odd numbers.

---

# Best Practices Followed by Experienced Developers

- Use meaningful expressions.
- Prefer parentheses for readability.
- Never confuse assignment and comparison.
- Be cautious with integer division.
- Understand short-circuit behavior.
- Use compound assignments to reduce repetition.
- Avoid overly complex logical conditions.
- Write conditions that are easy for humans to understand.

# C# BOOLEANS

---

# Why Do Booleans Exist?

Before learning:

```csharp
bool isLoggedIn = true;
bool hasPermission = false;
```

ask a deeper question:

> Why do computers need a special type that stores only `true` or `false`?

Because almost every program makes decisions.

Examples:

```text
Is the password correct?
Is the user an admin?
Has payment been completed?
Is the product in stock?
Did the operation succeed?
Is the player alive?
Is the form valid?
```

All of these questions have only two possible answers:

```text
Yes
No
```

or

```text
True
False
```

Without booleans:

```text
Programs could calculate,
but they could not decide.
```

Booleans exist because:

> Software must evaluate conditions and choose different behaviors based on those conditions.

```text
A light switch.
```

```text
ON
OFF
```

```text

```

```text

```

# What Is a Boolean?

A boolean is a data type that stores one of two values:

```text
true
false
```

In C#, the type is:

```csharp
bool
```

Example:

```csharp
bool isStudent = true;

bool hasLicense = false;
```

---

# Why Not Use Strings?

Why not do this?

```csharp
string status = "true";
```

Because strings can contain anything:

```text
"TRUE"
"true"
"yes"
"ok"
"banana"
```

The compiler cannot enforce correctness.

Booleans guarantee that only valid states exist.

---

# String vs Boolean

| Feature | string | bool |
|----------|---------|-------|
| Purpose | Store text | Store truth values |
| Possible Values | Unlimited | Only true/false |
| Type Safety | Low | High |
| Used for Decisions | Not Ideal | Yes |

# Logical AND (&&)

---

# What Is It?

Returns true only if BOTH conditions are true.

Example:

```csharp
bool result = true && true;

Console.WriteLine(result);
```

Output:

```text
True
```

---

# Truth Table

| A | B | A && B |
|---|---|----------|
| True | True | True |
| True | False | False |
| False | True | False |
| False | False | False |

---

# Real-World Usage

Example:

```csharp
bool canDrive =
    age >= 18 && hasLicense;
```

Meaning:

```text
Adult
AND
Has License
```

---

# Logical OR (||)

---

# What Is It?

Returns true if at least one condition is true.

Example:

```csharp
bool result = false || true;

Console.WriteLine(result);
```

Output:

```text
True
```

---

# Truth Table

| A | B | A || B |
|---|---|----------|
| True | True | True |
| True | False | True |
| False | True | True |
| False | False | False |

---

# Real-World Usage

Example:

```csharp

    isAdmin || isManager;
```

Meaning:

```text
Admin
OR
Manager
```

# Hidden Behavior: Short-Circuit Evaluation

One of the most important concepts.

---

# AND (&&)

Example:

```csharp
false && ExpensiveOperation()
```

Execution:

```text
false found
↓
Final result already false
↓
Second condition skipped
```

---

# OR (||)

Example:

```csharp
true || ExpensiveOperation()
```

Execution:

```text
true found
↓
Final result already true
↓
Second condition skipped
```

---

# Why Does This Exist?

Benefits:

```text
Performance
Safety
Efficiency
```

---

# Example

Safe code:

```csharp
if(name != null && name.Length > 0)
{
}
```

Why?

If:

```text
name == null
```

Then:

```text
name.Length
```

is never evaluated.

Avoids:

```text
NullReferenceException
```

# Interview Perspective

## What is a boolean?

Answer:

A boolean is a data type that stores one of two values: true or false.

---

## What is a boolean expression?

Answer:

An expression that evaluates to either true or false.

---

## Difference between = and ==?

Answer:

`=` assigns values, while `==` compares values.

---

## What is short-circuit evaluation?

Answer:

Logical operators `&&` and `||` stop evaluating once the result is already determined.

---

## Why use meaningful boolean names?

Answer:

They improve readability by expressing intent clearly.

---

## Are booleans integers in C#?

Answer:

No. C# does not allow implicit conversion between booleans and integers.

---

# Best Practices Followed by Experienced Developers

- Use descriptive boolean names.
- Prefer `if(isValid)` over `if(isValid == true)`.
- Take advantage of short-circuit evaluation.
- Keep boolean expressions readable.
- Extract complex conditions into well-named variables.
- Initialize local variables before use.
- Use TryParse when reading boolean input.
- Focus on expressing business intent.

---

# Final Mental Model

Think of booleans as the decision-making system of your program.

```text
Questions About Reality
↓
Comparison Operators
↓
Produce Booleans
↓
Logical Operators Combine Conditions
↓
true / false Decisions
↓
Program Chooses Behavior
↓
Software Appears Intelligent
```

A beginner sees:

```csharp
bool canAccess =
    isLoggedIn && hasPermission;
```

as syntax.

An experienced engineer sees:

> Business rules translated into explicit truth statements that capture intent, enable safe decision-making through short-circuit evaluation, and allow software to model the real-world conditions under which actions should or should not occur.

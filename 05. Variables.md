# C# VARIABLES

---

# Why Do Variables Exist?

Before understanding C# variables, understand the problem they solve.

Imagine trying to calculate a student's average marks without variables.

You somehow receive:

```text
85
90
78
```

Now imagine needing those values later.

Where do you keep them?

How do you refer to them?

How do you change them?

Without variables, programming would look like trying to solve a math problem by constantly rewriting numbers everywhere.

Example:

Without variables:

```csharp
Console.WriteLine((85 + 90 + 78) / 3);
```

Now suppose the marks change.

You must hunt down every occurrence of those numbers.

As software grows:

```text
Thousands of values
Millions of operations
Hundreds of developers
```

This becomes impossible.

Variables solve this problem.

> Variables give names to data so programs can remember, manipulate, and communicate information.

---

# What Is a Variable?

A variable is a named storage location that holds data.

Example:

```csharp
int age = 20;
```

Here:

```text
age  → Variable Name
int  → Type of data
20   → Value stored
```

---

# Mental Model

Think of variables as labeled boxes.

Example:

```text
+------------+
| Age        |
|------------|
| 20         |
+------------+
```

The label:

```text
Age
```

helps you find the box.

The contents:

```text
20
```

can be used later.

---

# Why Not Just Use Values Directly?

Consider:

```csharp
Console.WriteLine(20);
```

What does 20 mean?

Is it:

- Age?
- Salary?
- Discount percentage?
- Number of retries?

Nobody knows.

Now:

```csharp
int minimumVotingAge = 20;
```

Instant clarity.

Variables improve communication.

---

# What Problem Do Variables Solve?

Without variables:

## No Memory

Programs cannot remember values.

---

## No Reusability

Values must be rewritten repeatedly.

---

## Difficult Maintenance

Changing a value requires searching everywhere.

---

## Poor Readability

Magic numbers appear.

Example:

```csharp
price = amount * 18 / 100;
```

What is 18?

Now:

```csharp
int gstPercentage = 18;
price = amount * gstPercentage / 100;
```

Much clearer.

# Mental Model of Assignment

---

# Variables Are NOT Mathematical Equations

Mathematics:

```text
x = x + 1
```

Impossible.

How can something equal itself plus one?

Programming:

```csharp
x = x + 1;
```

Means:

```text
Take current value of x
Add 1
Store result back into x
```

---

# Example

Initial:

```csharp
int x = 5;
```

Memory:

```text
x → 5
```

Then:

```csharp
x = x + 1;
```

Step-by-step:

```text
Read x      → 5
Add 1       → 6
Store back  → x becomes 6
```

Final:

```text
x → 6
```

# C# CONSTANTS

---

# Why Do Constants Exist?

Imagine writing this:

```csharp
double area = 3.14159 * radius * radius;
```

Then later:

```csharp
circumference = 2 * 3.14159 * radius;
```

Then:

```csharp
volume = 4.0 / 3.0 * 3.14159 * radius * radius * radius;
```

Suppose someone accidentally changes one value:

```csharp
3.14
```

Now:

```text
Different formulas use different values of PI.
```

The application becomes inconsistent.

Constants solve this.

---

# What Is a Constant?

A constant is a named value whose value cannot change after it is assigned.

Syntax:

```csharp
const type name = value;
```

Example:

```csharp
const double PI = 3.14159;
```

---

# Mental Model

Variables:

```text
Reusable containers whose contents can change.
```

Constants:

```text
Permanent labels engraved on stone.
```

Example:

```text
Variable → Whiteboard
Constant → Monument
```

---

# Why Do Constants Exist?

They prevent accidental modification.

Without constants:

```csharp
double taxRate = 18;
taxRate = 25;
```

Was that intentional?

Nobody knows.

With constants:

```csharp
const double TAX_RATE = 18;
```

Compiler prevents changes.

---

# Example

```csharp
const int DAYS_IN_WEEK = 7;

Console.WriteLine(DAYS_IN_WEEK);
```

Output:

```text
7
```

---

# Attempting Modification

Incorrect:

```csharp
const int DAYS = 7;

DAYS = 10;
```

Compiler Error.

---

# Internal Working

Example:

```csharp
const int MAX_USERS = 100;
```

Compiler:

```text
Reads constant
↓
Ensures value assigned immediately
↓
Embeds value into compiled code
↓
Prevents reassignment
```

---

# Why Must Constants Be Initialized?

Incorrect:

```csharp
const int MAX_USERS;
```

Compiler Error.

---

# Why?

Because constants must be known during compilation.

They cannot be empty.

---

# When Should You Use Constants?

Use for values that truly never change.

Examples:

```csharp
const int MONTHS_IN_YEAR = 12;
const double PI = 3.14159;
const string COMPANY_NAME = "ABC Ltd";
```

---

# When Should You Avoid Constants?

Avoid when values may change.

Bad:

```csharp
const decimal GST = 18;
```

What if tax regulations change?

A configuration file may be better.

---

# Advantages

- Prevent accidental changes
- Improve readability
- Eliminate magic numbers
- Enable compiler optimizations

---

# Disadvantages

- Require compile-time values
- Not suitable for changing business rules

---

| Initialization | Optional initially | Mandatory |
| Reassignment | Allowed | Not Allowed |

Easier to read.

Less error-prone.

# Why Declare Multiple Variables?

Imagine:

```csharp

int y = 20;
int z = 30;
```

Many related values often need declaration.

C# provides concise syntax.

---

# Declaring Multiple Variables

Example:

```csharp

```

Equivalent to:

```csharp
int x = 5;
int y = 10;
int z = 15;
```

---

# Mental Model

Think of it as ordering multiple boxes of the same type at once.

Instead of:

```text
Box
Box
Box
```

You say:

```text
Give me three integer boxes.
```

---

# Internal Working

Compiler interprets:

```csharp
int x = 5, y = 10;
```

as:

```text
Declare x
Initialize x
Declare y
Initialize y
```

Separately.

---

# Declaring Without Initialization

Example:

```csharp
int x, y, z;
```

Valid.

But:

```csharp
Console.WriteLine(x);
```

Compiler Error.

Local variables must be initialized before use.

---

# Assigning Same Value

Example:

```csharp
int x, y, z;

x = y = z = 50;
```

Result:

```text
x → 50
y → 50
z → 50
```

---

# Internal Working

Execution:

```text
Assign 50 to z
↓
Assign z's value to y
↓
Assign y's value to x
```

---

# Advantages

- Less repetitive code
- Cleaner declarations
- Convenient for related values

---

# Disadvantages

- Can reduce readability if overused

Bad:

```csharp
int a = 1, b = 2, c = 3, d = 4, e = 5;
```

Harder to scan.

---

# Best Practice

Use multiple declarations sparingly.

Prefer clarity.

# C# CLASSES AND OBJECTS

---

# Why Do Classes and Objects Exist?

Before learning syntax like:

```csharp
class Car
{
}

Car myCar = new Car();
```

ask a deeper question:

> Why were classes and objects invented?

Imagine building a school management system without classes.

You need to represent:

```text
Students
Teachers
Courses
Exams
Attendance
Payments
```

Without classes:

```csharp
string student1Name;
int student1Age;

string student2Name;
int student2Age;

string teacher1Name;
string teacher1Subject;
```

Problems:

```text
Data becomes scattered.
No grouping of related information.
Code duplication explodes.
Maintenance becomes painful.
Large systems become chaotic.
```

Now imagine building:

```text
Amazon
Instagram
Netflix
Banking Software
Hospital Systems
Games
```

Without classes:

> Organizing software would become nearly impossible.

Classes and objects exist because:

> Real-world systems contain entities with data and behavior, and software needs a way to model them naturally.

# What Is a Class?

A class is:

> A blueprint or template used to define the structure and behavior of objects.

It describes:

```text
What data objects have.
What actions objects can perform.
```

---

# What Is an Object?

An object is:

> A real instance created from a class.

Example:

Class:

```text
Car
```

Objects:

```text
BMW
Tesla
Audi
Toyota
```

# Classes Are Reference Types

One of the most important concepts.

---

# Mental Model


```text

↓

↓
```


# Why?

Because:

```text
car1
```

| Count | Usually one | Many |

# Common Beginner Mistakes

---

# Mistake 1: Confusing Class and Object

Wrong:

```text
They are the same thing.
```

Reality:

```text
Class → Blueprint

Object → Real instance
```

---

# Mistake 2: Forgetting new

Incorrect:

```csharp
Car car;

car.brand = "Tesla";
```

Error:

```text
No object created.
```

---

# Mistake 3: Assuming Assignment Copies Objects

Example:

```csharp
car2 = car1;
```

Reality:

```text
Reference copied.
Not object.
```

---

# Mistake 4: Leaving Fields Public Everywhere

Works for learning.

Not ideal in production.

Use:

```text
Properties
Encapsulation
```

---

# Mistake 5: Ignoring Constructors

Can lead to invalid objects.

---

# Hidden Behaviors and Edge Cases

---

# Classes Are Reference Types

# Objects Live on the Heap

Reference variables typically live elsewhere (stack/local scope).

---

# Multiple References Can Point to Same Object

Shared state.

---

# Fields Get Default Values

Even without initialization.

---

# Unused Objects Become Eligible for Garbage Collection

Example:

```text
No references remain

Garbage Collector frees memory
```

# Interview Perspective

## What is a class?

Answer:

A class is a blueprint that defines the data and behavior of objects.

---

## What is an object?

Answer:

An object is an instance created from a class.

---

## Difference between class and object?

Answer:

A class defines structure, while an object is a real entity created from that structure.

---

## Are classes value types or reference types?

Answer:

Classes are reference types.

---

## What does new do?

Answer:

It creates an object in memory and returns a reference to it.

---

## What are fields?

Answer:

Fields are variables that store an object's data.

---

## Why are classes important?

Answer:

They allow software to model real-world entities in an organized, reusable, and maintainable way.

---

# Best Practices Followed by Experienced Developers

- Design classes around real responsibilities.
- Keep related data and behavior together.
- Avoid exposing fields directly; prefer properties.
- Use constructors to create valid objects.
- Remember that classes are reference types.
- Give classes meaningful names.
- Follow the Single Responsibility Principle.
- Model the domain naturally.

---

# Final Mental Model

Think of object-oriented programming as modeling the real world.

```text
Real-World Entity
↓
Create Class Blueprint
↓
Define Data (Fields)
↓
Define Behavior (Methods)
↓
Use new to Build Objects
↓
Objects Store State
↓
Objects Perform Actions
↓
Multiple Objects Collaborate
↓
Complex Software Emerges
```

A beginner sees:

```csharp

```

as syntax.

An experienced engineer sees:

> The creation of a real software entity from an abstract blueprint, allocating memory, encapsulating state and behavior, and enabling large systems to be built by modeling the problem domain through interacting objects.

# C# LOOPS

---

# Why Do Loops Exist?

Before learning:

```csharp
for
while
do...while
foreach
```

ask a deeper question:

> Why do programming languages need loops?

Imagine a world without loops.

Suppose you want to print:

```text
Hello
```

100 times.

Without loops:

```csharp
Console.WriteLine("Hello");
Console.WriteLine("Hello");
Console.WriteLine("Hello");
Console.WriteLine("Hello");
...
(repeated 100 times)
```

Problems:

```text
Huge code duplication
Hard to maintain
Error-prone
Not scalable
Impossible for unknown repetition counts
```

Now imagine:

```text
Read 10,000 records from a database
Process every employee
Validate every character in a password
Display every product in an e-commerce site
Calculate total marks for every student
```

Without loops:

> Most software we use today would not exist.

Loops exist because:

> Programs often need to repeat actions efficiently until some condition is met.

# What Is a Loop?

A loop is a control structure that repeatedly executes a block of code.

It continues until:

```text
A condition becomes false
OR
The collection ends
OR
The loop is explicitly stopped
```

---

# Why Not Just Copy and Paste?

Because loops provide:

```text
Automation
Maintainability
Scalability
Efficiency
Flexibility
```

---

# Types of Loops in C#

C# provides four primary loops:

| Loop | Best Used When |
|--------|----------------|
| `for` | Number of iterations is known |
| `while` | Repetition depends on a condition |
| `do...while` | Code must run at least once |
| `foreach` | Iterating collections |

---


Example:

```text
Start
↓
```

# FOR LOOP

---

# Why Does For Exist?

For loops solve problems where:

> You know how many times something should happen.

Examples:

```text
Print numbers 1–100
Process 12 months
Run 5 retries
Display top 10 products
```

---

# Syntax

```csharp
for(initialization; condition; update)
{
    // code
}
```

---

# Example

```csharp
for(int i = 1; i <= 5; i++)
{
    Console.WriteLine(i);
}
```

Output:

```text
1
2
3
4
5
```

---

# Breaking Down the For Loop

```csharp
for(int i = 1; i <= 5; i++)
```

Initialization:

```csharp
int i = 1;
```

Runs once.

---

Condition:

```csharp
i <= 5
```

Determines whether to continue.

---

Update:

```csharp
i++
```

Changes the loop state.

---

# Step-by-Step Execution

```text
i = 1
↓
1 <= 5 ? Yes
↓
Print 1
↓
i++
↓
i = 2
↓
Repeat
```

Until:

```text
6 <= 5 ? No
↓
Exit Loop
```

---

# Mental Model

Think:

```text
A countdown timer.

Start value
↓
Check limit
↓
Perform task
↓
Move closer to finish
```

# WHILE LOOP

---

# Why Does While Exist?

Sometimes you DON'T know how many times repetition is needed.

Examples:

```text
Wait until user enters valid input
Read data until end of file
Keep retrying connection
Run game until player quits
```

---

# Syntax

```csharp
while(condition)
{
    // code
}
```

---

# Example

```csharp

while(count <= 5)
{
    Console.WriteLine(count);
    count++;
}
```

Output:

```text
1
2
3
4
5
```

---

```text

↓

↓
Execute Block
↓
Update State
↓
Condition?
```

Execution:

```text

```

# WHILE vs DO...WHILE

| Feature | while | do...while |
|----------|--------|------------|
| Condition Checked | Before | After |
| Minimum Executions | 0 | 1 |
| Use Case | Conditional repetition | Must execute once |

# FOREACH LOOP

---

# Why Does Foreach Exist?

Developers frequently iterate collections.

Examples:

```text
Products
Employees
Students
Orders
Characters in a string
```

Using indexes repeatedly becomes tedious.

---

# Syntax

```csharp
foreach(type item in collection)
{
    // code
}
```

---

# Example

```csharp
string[] fruits =
{
    "Apple",
    "Banana",
    "Mango"
};

foreach(string fruit in fruits)
{
    Console.WriteLine(fruit);
}
```

Output:

```text
Apple
Banana
Mango
```

---

# Mental Model

Think:

```text
Give me each item,
one at a time.
```

---

# Internal Working

Reality:

```text
Collection
↓
Enumerator Created
↓
Move Next
↓
Retrieve Current Item
↓
Repeat Until Finished
```

---

# Advantages

```text
Cleaner syntax
Safer iteration
No index mistakes
Easy to read
```

---

# Disadvantages

```text
No direct index access
Cannot easily modify collection structure
Less control
```

# BREAK STATEMENT

---

# What Is Break?

Immediately exits the loop.

---

# Example

```csharp
for(int i = 1; i <= 10; i++)
{
    if(i == 5)
    {
        break;
    }

    Console.WriteLine(i);
}
```

Output:

```text
1
2
3
4
```

---

# Mental Model

Think:

```text
Emergency stop button.
```

---

# CONTINUE STATEMENT

---

# What Is Continue?

Skips the current iteration.

---

# Example

```csharp
for(int i = 1; i <= 5; i++)
{
    if(i == 3)
    {
        continue;
    }

    Console.WriteLine(i);
}
```

Output:

```text
1
2
4
5
```

---

# Mental Model

Think:

```text
Skip this item.
Continue working.
```

# Nested Loops

---

# What Are Nested Loops?

A loop inside another loop.

---

# Example

```csharp
for(int i = 1; i <= 3; i++)
{
    for(int j = 1; j <= 2; j++)
    {
        Console.WriteLine($"{i}, {j}");
    }
}
```

Output:

```text
1, 1
1, 2
2, 1
2, 2
3, 1
3, 2
```

---

# Mental Model

Think:

```text
Outer Loop
↓
For each outer iteration
↓
Run entire inner loop
```

# Hidden Danger

Example:


# Interview Perspective

## What is a loop?

Answer:

A loop repeatedly executes a block of code until a stopping condition is reached.

---

## Difference between for and while?

Answer:

`for` is preferred when the number of iterations is known, while `while` is used when repetition depends on a condition.

---

## Difference between while and do...while?

Answer:

`while` checks the condition before execution, whereas `do...while` executes at least once before checking the condition.

---

## What is foreach used for?

Answer:

It is used to iterate through collections in a simpler and safer way.

---

## What does break do?

Answer:

It immediately exits the current loop.

---

## What does continue do?

Answer:

It skips the current iteration and proceeds to the next one.

---

## What is an infinite loop?

Answer:

A loop that never terminates because its exit condition is never reached.

---

# Best Practices Followed by Experienced Developers

- Choose the loop that best expresses intent.
- Prefer `foreach` for collection traversal.
- Avoid unnecessary nested loops.
- Watch for off-by-one errors.
- Ensure loop conditions eventually become false.
- Use `break` and `continue` sparingly and intentionally.
- Keep loop bodies focused and readable.
- Think about performance when processing large datasets.

# C# VARIABLES

---

# Why Do Variables Exist?

Before understanding C# variables, understand the problem they solve.

Imagine trying to calculate a student's average marks without variables.

You somehow receive:

```text
85
90
78
```

Now imagine needing those values later.

Where do you keep them?

How do you refer to them?

How do you change them?

Without variables, programming would look like trying to solve a math problem by constantly rewriting numbers everywhere.

Example:

Without variables:

```csharp
Console.WriteLine((85 + 90 + 78) / 3);
```

Now suppose the marks change.

You must hunt down every occurrence of those numbers.

As software grows:

```text
Thousands of values
Millions of operations
Hundreds of developers
```

This becomes impossible.

Variables solve this problem.

> Variables give names to data so programs can remember, manipulate, and communicate information.

---

# What Is a Variable?

A variable is a named storage location that holds data.

Example:

```csharp
int age = 20;
```

Here:

```text
age  → Variable Name
int  → Type of data
20   → Value stored
```

---

# Mental Model

Think of variables as labeled boxes.

Example:

```text
+------------+
| Age        |
|------------|
| 20         |
+------------+
```

The label:

```text
Age
```

helps you find the box.

The contents:

```text
20
```

can be used later.

---

# Why Not Just Use Values Directly?

Consider:

```csharp
Console.WriteLine(20);
```

What does 20 mean?

Is it:

- Age?
- Salary?
- Discount percentage?
- Number of retries?

Nobody knows.

Now:

```csharp
int minimumVotingAge = 20;
```

Instant clarity.

Variables improve communication.

---

# What Problem Do Variables Solve?

Without variables:

## No Memory

Programs cannot remember values.

---

## No Reusability

Values must be rewritten repeatedly.

---

## Difficult Maintenance

Changing a value requires searching everywhere.

---

## Poor Readability

Magic numbers appear.

Example:

```csharp

```

What is 18?

Now:

```csharp
int gstPercentage = 18;
price = amount * gstPercentage / 100;
```

Much clearer.

# Mental Model of Assignment

Beginners often misunderstand assignment.

Example:

```csharp
int age = 25;
```

Many read:

> age equals 25

A better interpretation:

> Put 25 into the box named age.

---

# Variables Are NOT Mathematical Equations

Mathematics:

```text
x = x + 1
```

Impossible.

How can something equal itself plus one?

Programming:

```csharp
x = x + 1;
```

Means:

```text
Take current value of x
Add 1
Store result back into x
```

---

# Example

Initial:

```csharp
int x = 5;
```

Memory:

```text
x → 5
```

Then:

```csharp
x = x + 1;
```

Step-by-step:

```text
Read x      → 5
Add 1       → 6
Store back  → x becomes 6
```

Final:

```text
x → 6
```

# Variable Reassignment

Example:

```csharp

```

Initial:

```text

```

After reassignment:

```text

```

The old value is replaced.

# Variable Naming Rules

C# enforces rules.

---

# Rule 1: Must Start Properly

Allowed:

```csharp
age
Age
_age
studentCount
```

Not allowed:

```csharp
1age
@#
-name
```

---

# Rule 2: Cannot Use Spaces

Invalid:

```csharp
student age
```

Valid:

```csharp
studentAge
student_Age
```

---

# Rule 3: Cannot Use Reserved Keywords

Invalid:

```csharp
int class = 10;
```

Because:

```csharp
class
```

already has meaning.

---

# Rule 4: Case Sensitive

These are different:

```csharp
age
Age
AGE
```

Example:

```csharp
int age = 20;
int Age = 30;
```

Valid.

But confusing.

---

# Best Naming Practices

Bad:

```csharp

```

Good:

```csharp

```

# Interview Perspective

## What is a variable?

Answer:

A variable is a named storage location used to hold data that can be accessed and modified during program execution.

---

## Why are variables important?

Answer:

They allow programs to remember information, improve readability, reduce duplication, and support dynamic behavior.

---

## What is variable initialization?

Answer:

Assigning the first value to a variable.

Example:

```csharp
int age = 20;
```

---

## What is reassignment?

Answer:

Changing an existing variable's value.

Example:

```csharp
age = 25;
```

---

## Why is C# strongly typed?

Answer:

Because every variable has a defined type, allowing the compiler to catch errors early and improve reliability.

---

## Difference between declaration and initialization?

Answer:

Declaration creates the variable.

Initialization gives it its first value.

---

# Best Practices Followed by Experienced Developers

- Use meaningful names.
- Initialize variables as close as possible to their usage.
- Choose appropriate data types.
- Avoid magic numbers.
- Prefer readability over short names.
- Use constants for fixed values.
- Never sacrifice clarity for brevity.
- Let variable names communicate intent.

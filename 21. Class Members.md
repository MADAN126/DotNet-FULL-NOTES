# C# CLASS MEMBERS

---

# Why Do Class Members Exist?

Before learning syntax like:

```csharp
class Car
{
    public string Brand;
    public void Start()
    {
    }
}
```

ask a deeper question:

> Why do classes contain "members" at all?

Imagine creating a class like this:

```csharp
class Car
{
}
```

Question:

```text
What can this car do?
What information does it store?
How can we interact with it?
```

Answer:

```text
Nothing.
```

It is just an empty blueprint.

Real-world entities contain:

```text
Data
Behavior
Initialization rules
State
Shared information
```

Software entities need the same.

Class members exist because:

> A class needs building blocks to describe what an object knows, what it can do, and how it behaves.

---

# Mental Model

Think of a class as a company.

A company has:

```text
Employees
Policies
Departments
Responsibilities
Shared resources
```

Similarly, a class contains members.

---

# Visual Representation

```text
CLASS
(Employee)

Members:

Fields       → Data
Properties   → Controlled Data Access
Methods      → Actions
Constructors → Initialization
Constants    → Fixed Values
Events       → Notifications
Nested Types → Related Types
```

---

# What Are Class Members?

Class members are:

> The components defined inside a class.

They determine:

```text
What data the class stores.
What actions it performs.
How it is initialized.
How others interact with it.
```

---

# Types of Class Members

The most important members are:

```text
Fields
Properties
Methods
Constructors
Constants
Static Members
Events
Nested Classes
```

For beginners, focus heavily on:

```text
Fields
Methods
Constructors
Properties
Static Members
Constants
```

---

# 1. Fields

---

# What Are Fields?

Fields are variables declared inside a class.

They store an object's data.

---

# Why Do Fields Exist?

Objects need memory.

Example:

A student needs:

```text
Name
Age
Roll Number
```

A car needs:

```text
Brand
Speed
Fuel
```

---

# Syntax

```csharp
class Car
{
    public string brand;
    public int speed;
}
```

---

# Example

```csharp
Car car = new Car();

car.brand = "Tesla";
car.speed = 120;
```

---

# Memory Representation

```text
car
 ↓

------------------
brand = Tesla
speed = 120
------------------
```

---

# Advantages

```text
Simple
Fast access
Easy to understand
```

---

# Disadvantages

Public fields expose internal data directly.

---

# Beginner Mistake

Using public fields everywhere.

Example:

```csharp
car.speed = -500;
```

Invalid state.

---

# Best Practice

Prefer properties over public fields.

---

# 2. Methods

---

# What Are Methods?

Methods define behavior.

They specify:

> What an object can do.

---

# Why Do Methods Exist?

Objects are not just data containers.

A bank account should:

```text
Deposit()
Withdraw()
Transfer()
```

A car should:

```text
Start()
Stop()
Brake()
```

---

# Syntax

```csharp
class Car
{
    public void Honk()
    {
        Console.WriteLine("Beep!");
    }
}
```

---

# Example

```csharp
Car car = new Car();

car.Honk();
```

Output:

```text
Beep!
```

---

# Mental Model

Think:

```text
Fields
↓
What object knows

Methods
↓
What object does
```

---

# Advantages

```text
Encapsulates logic
Improves readability
Encourages reuse
```

---

# Beginner Mistake

Creating methods that do too many things.

Bad:

```text
ProcessOrderAndSendEmailAndUpdateInventory()
```

---

# Best Practice

One method.

One responsibility.

---

# 3. Constructors

---

# What Are Constructors?

Constructors initialize objects.

They run automatically when an object is created.

---

# Why Do Constructors Exist?

Without constructors:

Objects may start invalid.

Example:

```csharp
Car car = new Car();

car.brand = null;
```

---

# Constructors Ensure Valid State

Example:

```csharp
class Car
{
    public string brand;

    public Car()
    {
        brand = "Unknown";
    }
}
```

---

# Syntax

```csharp
public ClassName()
{
}
```

---

# Example

```csharp
Car car = new Car();

Console.WriteLine(car.brand);
```

Output:

```text
Unknown
```

---

# Internal Execution

```text
new Car()
↓
Allocate memory
↓
Initialize fields
↓
Run constructor
↓
Return reference
```

---

# Advantages

```text
Objects start correctly
Reduces errors
Improves consistency
```

---

# Beginner Mistake

Forgetting constructors exist.

---

# 4. Properties

---

# What Are Properties?

Properties provide controlled access to data.

---

# Why Do Properties Exist?

Public fields allow unrestricted modification.

Bad:

```csharp
account.balance = -100000;
```

---

# Properties Add Protection

Example:

```csharp
class Account
{
    private decimal balance;

    public decimal Balance
    {
        get { return balance; }

        set
        {
            if (value >= 0)
            {
                balance = value;
            }
        }
    }
}
```

---

# Mental Model

Think:

```text
Fields
↓
Storage

Properties
↓
Gatekeepers
```

---

# Advantages

```text
Validation
Encapsulation
Flexibility
Safer design
```

---

# Best Practice

Use properties instead of public fields.

---

# 5. Constants

---

# What Are Constants?

Constants store fixed values.

---

# Why Do They Exist?

Some values should never change.

Examples:

```text
PI
Tax Rate
Maximum Attempts
Company Name
```

---

# Syntax

```csharp
public const double PI = 3.14159;
```

---

# Example

```csharp
Console.WriteLine(MathCircle.PI);
```

---

# Advantages

```text
Readability
Safety
Eliminates magic numbers
```

---

# Beginner Mistake

Using variables for values that never change.

---

# Best Practice

Use constants for true constants.

---

# 6. Static Members

---

# What Are Static Members?

Static members belong to the class itself.

Not individual objects.

---

# Why Do They Exist?

Some data is shared.

Example:

Company Name:

```text
Same for all employees.
```

---

# Syntax

```csharp
public static int Count;
```

---

# Example

```csharp
class Employee
{
    public static int TotalEmployees;
}
```

Access:

```csharp
Employee.TotalEmployees++;
```

---

# Mental Model

Instance Member:

```text
Belongs to object.
```

Static Member:

```text
Belongs to blueprint.
```

---

# Visual Representation

Instance:

```text
employee1.age
employee2.age
```

Static:

```text
Employee.CompanyName
```

---

# Advantages

```text
Shared data
Utility methods
Memory efficient
```

---

# Disadvantages

Can create global state problems.

    }

To group closely related types.

| Member | Purpose | Belongs To |
|----------|----------|------------|
| Field | Store data | Object |

| Method | Perform actions | Object/Class |

# Interview Perspective

## What are class members?

Answer:

The components declared inside a class that define its data and behavior.

---

## Name common class members.

Answer:

Fields, properties, methods, constructors, constants, static members, events, and nested types.

---

## Difference between fields and properties?

Answer:

Fields store data directly, while properties provide controlled access to that data.

---

## Why use constructors?

Answer:

To initialize objects and ensure they start in a valid state.

---

## What are static members?

Answer:

Members that belong to the class itself rather than individual objects.

---

## Why avoid public fields?

Answer:

They expose internal state and allow invalid modifications.

---

# Best Practices Followed by Experienced Developers

- Prefer properties over public fields.
- Keep methods focused on one responsibility.
- Use constructors to enforce valid state.
- Use constants for fixed values.
- Use static members only for truly shared behavior.
- Encapsulate internal details.
- Design classes around real responsibilities.
- Avoid turning classes into "God Objects."

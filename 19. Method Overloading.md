# C# METHOD OVERLOADING

---

# Why Does Method Overloading Exist?

Before learning syntax like:

```csharp
Add(10, 20);
Add(10.5, 20.5);
Add(10, 20, 30);
```

ask a deeper question:

> Why would multiple methods have the same name?

Imagine method overloading did NOT exist.

Suppose you want to perform addition.

Without overloading:

```csharp
AddIntegers(int a, int b)

AddDoubles(double a, double b)

AddThreeIntegers(int a, int b, int c)

AddDecimals(decimal a, decimal b)
```

Problems:

```text
Too many method names
Harder to remember APIs
Reduced readability
Poor developer experience
```

Now imagine frameworks like .NET.

Without overloading:

```text
Thousands of slightly different method names.
```

Programming would become frustrating.

Method overloading exists because:

> One logical operation may need to work with different kinds of inputs.

---

```text
One employee.
Multiple ways to contact them.
```

Example:

```text
Call by phone
Email
Video meeting
Office visit
```

Same person.

Different methods of interaction.

Similarly:

```text
Add(int, int)

Add(int, int, int)
```

Same concept:

```text
Addition
```

Different input forms.

# What Is Method Overloading?

Method overloading means:

> Defining multiple methods with the same name but different parameter lists.

---

# Key Idea

The method name stays the same.

The parameters change.

---

# Example

```csharp
static int Add(int a, int b)
{
    return a + b;
}

static double Add(double a, double b)
{
    return a + b;
}
```

Both methods are called:

```text
Add
```

But they are different methods.

---

# Why Is This Useful?

Because developers think in terms of:

```text
Actions
```

not

```text
Data types.
```

Example:

Instead of:

```text
PrintInteger
PrintString
PrintDouble
```

we prefer:

```text
Print(...)
```

because the intention is:

```text
Printing.
```

# Types of Overloading

---

# 1. Different Number of Parameters

Example:

```csharp
static void Show()
{
    Console.WriteLine("No parameters");
}

static void Show(string name)
{
    Console.WriteLine(name);
}

static void Show(string name,
                 int age)
{
    Console.WriteLine(name);
    Console.WriteLine(age);
}
```

---

# Calling

```csharp
Show();

Show("John");

Show("John", 25);
```

---

# Compiler Decision

```text
Argument Count
↓
Choose Matching Method
```

---

# 2. Different Parameter Types

Example:

```csharp
static void Display(int number)
{
    Console.WriteLine("Integer");
}

static void Display(double number)
{
    Console.WriteLine("Double");
}
```

---

Calling:

```csharp
Display(5);

Display(5.5);
```

Output:

```text
Integer
Double
```

---

# Compiler Decision

```text
Argument Types
↓
Choose Matching Method
```

---

# 3. Different Parameter Order

Example:

```csharp
static void Print(string name,
                  int age)
{
}

static void Print(int age,
                  string name)
{
}
```

Both are valid.

---

Calling:

```csharp
Print("John", 25);

Print(25, "John");
```

---

# Why?

Because the signatures differ.

---

# What Is a Method Signature?

One of the most important concepts.

A method signature consists of:

```text
Method Name
+
Parameter Count
+
Parameter Types
+
Parameter Order
```

---

# Example

Signature:

```csharp
Add(int, int)
```

Different signature:

```csharp
Add(double, double)
```

Different signature:

```csharp
Add(int, int, int)
```

---

# What Is NOT Part of the Signature?

Many beginners misunderstand this.

---

# Return Type Is NOT Included

Incorrect:

```csharp
int Test()
{
    return 1;
}

double Test()
{
    return 1.5;
}
```

Compilation Error.

---

# Why?

Because:

```text
Test();
```

doesn't tell the compiler which one you want.

---

# Visual Representation

Compiler Sees:

```text
Test()
```

Choices:

```text
Test() → int
Test() → double
```

Ambiguous.

---

# Hidden Rule

You CANNOT overload methods using only return types.

---

# Method Overloading vs Method Overriding

A very common interview question.

---

# Method Overloading

```text
Same class
Same name
Different parameters
Compile-time polymorphism
```

---

# Method Overriding

```text
Parent and child classes
Same signature
Different implementation
Runtime polymorphism
```

---

# Comparison Table

| Feature | Overloading | Overriding |
|-----------|------------|-------------|
| Same Method Name | Yes | Yes |
| Same Class | Usually Yes | No |
| Parameters Change | Yes | No |
| Return Type Change | No | Limited |
| Polymorphism Type | Compile Time | Runtime |
| Requires Inheritance | No | Yes |

# Why Is It Called Compile-Time Polymorphism?

Because:

```text
Decision happens BEFORE execution.
```

The compiler already knows:

```text

```

# Implicit Type Conversion and Overloading

Hidden behavior many beginners miss.

Example:

```csharp

{
    Console.WriteLine("Double");
}

Show(5);
```

Output:

```text

```

---

# Why?

Because:

```text
int
↓
implicitly converts to
double
```

---

# Best Match Principle

The compiler prefers:

```text
Exact match
↓
Implicit conversion
↓
Other possibilities
```

---

# Example

```csharp

{
    Console.WriteLine("Int");
}

static void Test(double x)
{
    Console.WriteLine("Double");
}

Test(5);
```

Output:

```text
Int
```

---

# Why?

Because:

```text

```

is a better match than:

```text
double
```

# Why?

No clear winner.

# Interview Perspective

## What is method overloading?

Answer:

Method overloading is defining multiple methods with the same name but different parameter lists.

---

## What forms a method signature?

Answer:

Method name, parameter count, parameter types, and parameter order.

---

## Can methods be overloaded by changing only return types?

Answer:

No.

Return types are not part of the method signature.

---

## Why is overloading called compile-time polymorphism?

Answer:

Because the compiler determines which method to invoke before the program runs.

---

## Difference between overloading and overriding?

Answer:

Overloading uses different parameter lists within the same class, while overriding changes inherited behavior using the same signature.

---

## Give a real-world example of overloading.

Answer:

`Console.WriteLine()` supports many overloads for different data types.

---

# Best Practices Followed by Experienced Developers

- Overload methods only when they represent the same logical action.
- Keep overload behavior consistent.
- Avoid creating too many overloads.
- Prefer clarity over cleverness.
- Be cautious with default parameters and overload combinations.
- Consider named arguments for readability.
- Design APIs around intent, not data types.
- Document overload differences clearly.

# Final Mental Model

Think of method overloading as:

```text
One Action
↓
Multiple Ways To Perform It
↓
Compiler Examines Arguments
↓
Finds Best Matching Signature
↓
Calls Correct Method
```

Example:

```text
Add(5, 3)
↓
Add(int, int)

Add(5.5, 3.2)
↓
Add(double, double)

Add(1, 2, 3)
↓
Add(int, int, int)
```

A beginner sees:

```csharp
Add(10, 20);
```

as simply calling a method.

An experienced engineer sees:

> A compile-time polymorphism mechanism that allows APIs to express a single intent through multiple signatures, improving readability, usability, and flexibility while preserving strong type safety.
